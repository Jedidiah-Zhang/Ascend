
# 物品系统 — 设计

参照 Minecraft 的简洁物品模型，根据本游戏需求做适配。

## 物品模型基础

- 每个物品有唯一 ID
- 物品有两个物理属性：**重量**（kg）和**体积**（L）
- 同类物品可以无限堆叠，但受重量和体积限制——不是格子数限制，是物理容量限制
- 重量和体积任意一项超出承载上限即不可再装入

## 物品品类

### 工具

- 有耐久度，使用消耗耐久，耐久归零直接报废（不可恢复）
- 耐久用完**前**可以修理，消耗对应材料回复耐久
- 修理次数不限，只要定期维护就可以一直用

### 食物

- 提供饱腹值和水合值
- 携带消化标签——不同的 NPC 因消化基因不同，对同一食物的吸收率不同
- 有保质期（标签系统管理——"新鲜""变质"）

### 建材

- 高堆叠上限，用于建造设施和地形改造
- 不同建材决定建筑的强度、隔热、美观等属性

### 基因样本

- 基因样本不是独立携带的物品——玩家在野外获取的是**生物样本**（身体部位或活体），带到研究室后进入研究界面提取基因
- 生物样本各自占一格（不可堆叠），活体有存活时间限制
- 提取出的基因在研究室界面内操作——查看、编辑、植入——不进入背包
- 基因操作的产物（编辑好的基因，用于编辑或植入等）如果暂时不使用，可以存储于特定的基因储存设施中。

### 资源

- 原材料（木材、石材、纤维、矿物等），用于制作和建造
- 资源**不进入人物背包**——需要建设仓库存储，通过生物或载具搬运
- 采集后资源以"堆"的形式存在于地面上，需搬运到仓库才算入库

### 假肢/装备

- 可替换的身体部件，装在身体部位槽位上
- 有耐久（可修理），可升级，可携带原部位没有的功能

## 携带与存储

### 人物携带

- 人物有**负重上限**和**携带体积上限**
- 携带：工具、食物、少量个人用品、生物样本
- 资源类物品（木材、石材、矿物等）不随身携带——太重太大，需搬运

### 仓库

- 聚落的储存设施，重量和体积上限远大于人物
- 建造和制作消耗的资源从仓库扣除
- 多个仓库之间可以通过运输网络联动

### 搬运

- 资源从采集点到仓库需要**搬运**——由驯化的生物或载具完成
- 生物和载具有各自的负重和体积容量
- 搬运路线、效率由经济物流模块管理
- 搬运过程中资源可能损耗（被袭击、天气损坏）

## 与其他模块的接口

- → **基因系统**：生物样本→研究室提取基因→基因操作界面，基因样本不进入常规背包
- → **生命个体**：食物消化标签 → 饮食兼容 → 不同 NPC 吸收率不同；假肢→身体部位槽位
- → **玩家行动**：工具用于采集、建造、制作；建材是建造的消耗品；搬运需要生物/载具（→畜牧驯化、经济物流）
- → **心智系统**：物品的质量/来源可作为 NPC 对话和评价的素材
