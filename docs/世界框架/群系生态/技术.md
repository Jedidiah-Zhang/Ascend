# 群系生态 — 技术

## 群系数据

- 每个群系类型一个 `BiomeTemplate`，定义：TerrainBias（地形偏移）、生态参数（water/mountain/tree_ratio）、生物列表（权重）、资源列表（权重）
- 分块生成时由 `biome_from_attrs()` 算主隶属群系，取模板实例化
- 16 陆地群系 + 3 海洋群系，枚举 `BiomeType`（int 0-18）

## 群系细分

每气候档 2 个子型，细分维度用连续场：

```python
@dataclass
class _SubdivConfig:
    dimension: str        # "rainfall"/"temperature"/"altitude"/"moisture"
    low: BiomeType        # 低端子型
    high: BiomeType       # 高端子型
    value_min: float      # 归一化下限（大陆场实际值域）
    value_max: float      # 归一化上限
    split: float = 0.5    # 分界点
```

`biome_membership(attrs)` 返回 `[(BiomeType, weight), ...]`，权重和为 1.0。三角形隶属函数：
```
c_lo=0.25, c_hi=0.75
w_lo = max(0, 1 - |v - c_lo| / 0.5)
w_hi = max(0, 1 - |v - c_hi| / 0.5)
归一化后 w_lo + w_hi = 1
```

边界处（v≈0.5）两权重各 0.5 → 平滑混合。

## TerrainBias — 群系对地形分类的偏移

```python
@dataclass(slots=True)
class TerrainBias:
    sand_cap_delta: float = 0.0       # SAND 海拔上限偏移
    fertile_shift: float = 0.0        # FERTILE_SOIL 海拔带平移
    rock_threshold_delta: float = 0.0 # ROCK 阈值偏移（只影响 ROCK）
    peak_threshold_delta: float = 0.0 # MOUNTAIN_PEAK 阈值偏移
    marsh_tendency: float = 0.0       # MARSH 倾向 [0,1]
```

基线 = `TEMPERATE_DECIDUOUS_FOREST`（全 0）。tile 生成时 `bias = Σ weight_i × bias_i`，数值字段加权平均。chunk 边界因连续场 → 隶属度连续 → bias 连续 → 地形连续。

`creature_weights` / `resource_weights` 不混合，按主隶属选单一模板。

## 微地形生成

- tile 生成时从层1宏观场采样 tile 级气候属性（温度/降雨/海拔/moisture）
- `biome_membership()` 算隶属度 → 混合 TerrainBias
- `_classify(elev, temp, bias)` 按偏移后海拔带分类 TerrainType
- 群系模板定义子区域类型和出现概率，叠加噪声细节决定微地形
- 子区域边界自然渐变（隶属度混合），不硬切

## 积雪

当前禁用（`_SNOW_ENABLED = False`）。MOUNTAIN_PEAK 只由海拔带决定。天气系统接入后改为动态雪线，积雪线不作为群系。

## 群系存在性

不强制保证子型存在。细分维度 value_min/value_max 基于大陆场实际值域（非理论极值），保证统计上大概率出现。稀有群系（出现但范围小）是重玩价值的来源。

诊断：跑多 seed 统计每个子型出现面积，确认无"理论存在但概率趋零"的设计失误。

## 生物种群

- 每个分块维护该群系下的生物种群计数（不追踪每个个体的实时位置）
- 离屏时种群按生态模型自周转（捕食-繁殖-死亡）
- 玩家进入时种群实例化为具体生物个体，分布到子区域

## 群系扩展

- 新增群系子型只需要：扩 `BiomeType` 枚举 + 写 `BiomeTemplate` + 更新 `_SUBDIV_CONFIGS`
- 不需要改 tile 生成代码（TerrainBias 自动生效）
