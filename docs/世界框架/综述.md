# 世界框架 — 综述

世界框架定义游戏的物理基础：空间、时间、物质、事件。

## 子模块关联

```
地图生成 → 群系生态 → 季节气候
                ↑          ↓
                └── 时间系统 ──┘
                 ↓
              物品系统
                 ↓
    ═══════ 事件系统 ═══════
         （底层状态变更协议）
```

- **地图生成**定义双层地图结构——大地图（无限延伸的分块网格，用于选点和远行规划）+ 详细地图（固定 200×200 tile，无缝拼接，用于活动/建造/社交）
- **群系生态**定义群系 = 气候 + 生态内容（地形、植物、生物、资源）。初始两个基础群系（温带落叶林、干旱灌木地），后续按需扩展
- **季节气候**定义气候（8 档离散分类，指导群系分布）+天气（连续值，六项可扩展参数：温度、降雨量、日照、海拔、湿度、风速）。解析算架构，四层叠加（年均基线+季节调制+昼夜偏移+大气扰动），降水为事件驱动模型。天气数据通过 8 种 per-parameter 事件发布到 WorldTree，不提供查询 API
- **时间系统**定义基底时钟（2h=1天）、短跳（睡觉/移动）和长跳（闭关）三种时间跨越，以及快进期间的世界模拟方式
- **物品系统**Minecraft 式简洁模型——格子系统 + 可堆叠/不可堆叠。六类物品：工具（耐久/修理/报废）、食物（饱腹+消化标签）、建材、基因样本（生物样本→研究室提取，基因不进入背包）、资源、假肢装备
- **事件系统**是连接所有模块的**总线**。各模块不直接耦合，所有变化（天气、NPC 行动、玩家操作）统一作为事件发布到总线上流转。事件之间形成有向图（因果、观测、传播等关系）。观测分感知层（自动物理判定）和注意层（主观关注程度）。事件总线是快进模拟的驱动骨架

## 与其他大类的接口

- → **生命个体**：环境参数（温度、湿度）作为生理需求的输入；物品（食物）满足饱腹
- → **群体社会**：季节影响资源供给，间接影响社会稳定性
- → **基因系统**：不同群系的生物携带不同基因，驱动探索动机
- → **玩家行动**：种植采集依赖季节和群系；威胁事件触发防御需求
- → **表现层**：地图、群系、季节的视觉呈现
- → **时间系统**：快进期间的人生轨迹摘要供心智系统 LLM 检索；世界时钟驱动生理需求衰减和世代更替
