# 季节气候 — 设计

## 两层模型

```
气候（离散档位）              天气（连续值）
─────────────────           ─────────────────
群系分类的依据               此刻的真实数值
指导天气范围                  作为生理/种植/基因的输入
季节嵌入气候内部              随时钟实时波动
```

- **气候**：离散档位（8 种 ClimateZone），描述一个区域的长期气象特征。用于群系分类、大致判断"这里冷不冷、干不干"
- **天气**：连续值，此刻的具体数值——温度是 23.7°C 而不是"温暖"。用于生理需求计算、作物生长判定、基因适应区间匹配

## 气象参数池

可扩展。初始六项：

| 参数 | 单位 | 说明 |
|---|---|---|
| 温度 | °C | 体感温度，影响生理需求（体温维持）、作物生长期 |
| 降雨量 | mm/年（基线）/ mm/小时（实时） | 基线决定气候档位；实时值为降雨强度，影响土壤湿度、洪涝事件 |
| 日照时长 | 小时/天 | 影响作物光合作用、某些生物的活动周期 |
| 海拔 | m | 决定气压（推导值），影响体能消耗和某些基因表达。同时是气候判定的第一优先级（≥2000m → 高山） |
| 湿度 | % | 相对湿度，影响腐烂速度、霉菌生长、体感舒适度 |
| 风速 | m/s | 影响体感温度、风暴事件、某些飞行生物的移动 |

海拔推导值：气压 = 1013.25 × (1 - 海拔/44330)^5.255，含氧量等比例变化。

## 四层叠加（解析算架构）

天气采用**解析算**（resolved computation）——无快照、无缓存，每游戏分钟从零计算：

```
年均基线（气候档位 + 噪声，chunk 生成时确定）
    +
季节偏移（余弦曲线，由 SeasonalityMode 控制形状）
    +
昼夜偏移（余弦曲线，峰值 14:00 / 谷值 02:00，湿度反转）
    +
大气扰动（2D Perlin 噪声场，沿风向漂移，空间连续）
    =
此刻天气
```

关键设计：
- **温度**：四层全部参与
- **湿度**：四层全部参与（季节偏移可受 SeasonalityMode 阶梯化；昼夜偏移反比于温度——夜间高湿、午间低湿）
- **风速**：基线 + 大气扰动（不参与季节/昼夜调制）
- **日照**：拆为两个独立维度——
  - **日照时长**（daylight_hours）：天文计算 `daylight_hours(day_of_year, latitude)`，日级值，季节+纬度驱动
  - **日照强度**（sunshine_intensity）：`sin((hour - sunrise) / daylight × π)` 正弦曲线，正午峰值，夜间=0，含降雨衰减 + 大气噪声微调
- **日照时长感知**按日长分级（极短/较短/适中/较长/很长/极长），**日照强度感知**按明暗分级（夜/昏暗/适中/明亮/强光）
- **降雨**：事件驱动（RainSchedule），非连续计算。降水类型由当前温度判定：>0°C 为雨，≤0°C 为雪

## 气候档位（8 种）

用于群系分类的离散标签。判定顺序（纯静态，由年均温+年降雨+海拔决定）：

| 优先级 | 条件 | 气候档位 | 季节模式 |
|---|---|---|---|
| 1 | 海拔 ≥ 2000m | 高山 (Alpine) | ALPINE |
| 2 | 年均温 < -5°C | 极地苔原 (Polar Tundra) | POLAR |
| 3 | 年降雨 < 200mm | 沙漠 (Desert) | NONE |
| 4 | 年降雨 < 600mm 且 T>5°C | 草原 (Steppe) | FOUR_SEASON |
| 5 | T≥20°C, R≥1500mm | 热带雨林 (Equatorial Rainforest) | NONE |
| 5 | T≥20°C, R<1500mm | 热带草原 (Tropical Savanna) | MONSOON |
| 6 | 5°C ≤ T < 20°C | 温带森林 (Temperate Forest) | FOUR_SEASON |
| 7 | -5°C ≤ T < 5°C, R≥400mm | 亚寒带针叶林 (Subarctic Taiga) | POLAR |
| 7 | -5°C ≤ T < 5°C, R<400mm | 极地苔原（合并入冷干） | POLAR |

温度季节振幅不再由气候档位离散决定，而是由 WeatherEngine 从年均温+年降雨连续推导（详见下文"季节"节），保证气候带交界处无跳变。

### 季节性模式（SeasonalityMode）

| 模式 | 气候带 | 温度曲线 | 湿度曲线 |
|---|---|---|---|
| NONE | 赤道雨林、沙漠 | 余弦，振幅极小 | 余弦（sharpness=0） |
| MONSOON | 热带草原 | 余弦 | **tanh 阶梯化**（sharpness=2.5），旱湿切换突兀 |
| FOUR_SEASON | 温带森林、草原 | 标准余弦 | 余弦（sharpness=0） |
| POLAR | 亚寒带、极地 | 余弦，振幅大 | 余弦（sharpness=0） |
| ALPINE | 高山 | 余弦 | 余弦（sharpness=0） |

## 季节

季节嵌入气候类型——不是独立的全局时钟切换。四季划分：
- 每年 360 天，每季 90 天
- 春=0, 夏=1, 秋=2, 冬=3
- 季节调制是连续余弦波——夏季中点（progress=1.5）取 +amplitude，冬季中点（progress=3.5）取 -amplitude，平滑过渡

温度季节振幅由年均温+年降雨连续推导（非气候档位离散取值）：
- 基础振幅随年均温线性变化：极地(-5°C)→28°C，赤道(35°C)→2°C
- 干旱修正：低降雨大陆性气候 +最多 4°C，高降雨海洋调节 -最多 2°C
- 钳制在 [1, 30]
- 保证气候带交界处无跳变（年均温/降雨连续 → 振幅连续）

## 昼夜

- 温度峰值 14:00（+amplitude），谷值 02:00（-amplitude）
- 湿度反转：14:00 最低湿，02:00 最高湿
- 昼夜振幅 = 季节振幅 × 0.5（昼夜变化小于季节变化）
- 日出/日落基于太阳赤纬 + 纬度公式计算，支持极昼/极夜边界
- **per-chunk 昼夜**：每个 chunk 用自己的纬度判定昼夜切换，发布 sunrise/sunset 事件（携带当日 `daylight_hours` 供下游种植/生理系统使用）。不同纬度 chunk 的日出日落时刻不同，极地极昼极夜单独体现

## 大气扰动场

- 2D Perlin 噪声，采样间距 2km（一个大气特征覆盖约 10×10 chunks）
- 采样坐标沿风向随时间漂移，模拟气团移动
- 风向由第二个低频 Perlin 驱动，缓慢旋转模拟季节性风向变化
- **相邻 chunk 空间连续**——无需跨 chunk 查询，O(1) 计算

## 降水模型（事件驱动）

降雨不是连续变化，而是**离散事件**：

```
一场雨 = ramp_up(前 15-35%) → peak(中间段) → ramp_down(后 15-35%)
```

- 从年降雨量反推事件频率：年降雨总小时 = R / 平均强度，事件数 = 总小时 / 平均持续，间隔 = (年小时 - 总小时) / 事件数
- 间隔/持续/强度在均值 0.5x-1.5x 范围内随机化（确定性 RNG，chunk 坐标派生种子）
- 预排未来 4 场雨，队列低于 2 场时补算
- **降水类型判定**：当前温度 > 0°C → 雨（rain），≤ 0°C → 雪（snow）

### 各气候带降雨档位

| 气候带 | 平均强度 (mm/h) | 平均持续 (h) | ramp 比例 |
|---|---|---|---|
| 热带雨林 | 10.0 | 2.0 | 0.15/0.15 |
| 热带草原 | 8.0 | 1.5 | 0.2/0.2 |
| 沙漠 | 2.0 | 0.5 | 0.35/0.35 |
| 草原 | 4.0 | 1.5 | 0.25/0.25 |
| 温带森林 | 5.0 | 2.0 | 0.2/0.2 |
| 亚寒带针叶林 | 3.0 | 1.5 | 0.2/0.25 |
| 极地苔原 | 2.0 | 1.0 | 0.25/0.3 |
| 高山 | 3.0 | 1.0 | 0.25/0.25 |

## 事件与查询（双通道）

天气数据通过两个互补通道对外提供：

### 粗粒度感知事件

事件只在**感知类别变化**时发布，不再按固定数值阈值。由角色的直观体感定义类别边界，而非气象学精度：

| 参数 | 级数 | 感知类别 |
|------|------|---------|
| 温度 | 10 级 | 严寒(-10) → 冰冷(-3) → 寒冷(5) → 微凉(13) → 凉爽(20) → 温和(25) → 温暖(30) → 炎热(36) → 酷热(43) → 极热 |
| 湿度 | 5 级 | 干燥(25) → 舒适(50) → 潮湿(72) → 非常潮湿(88) → 闷热 |
| 风速 | 6 级 | 无风(1.5) → 轻风(4) → 微风(8) → 大风(14) → 强风(23) → 狂风 |
| 日照时长 | 6 级 | 极短(1.5h) → 较短 → 适中 → 较长 → 很长 → 极长(>15.5h) |
| 日照强度 | 5 级 | 夜(0.01) → 昏暗(0.25) → 适中(0.55) → 明亮(0.80) → 强光 |

每个事件携带 `perception` 标签 + 精确数值，供 AI 决策（`if cold: seek_shelter`）的同时不必二次查询。

以下离散事件保持原逻辑不变：

| 事件类型 | 触发 | 携带数据 |
|---|---|---|
| `temperature_change` | 温度感知类别变化 | temperature, **perception**, season, time_of_day |
| `humidity_change` | 湿度感知类别变化 | humidity, **perception**, time_of_day |
| `wind_change` | 风速感知类别变化 | wind_speed, **perception**, wind_dir_x/y, time_of_day |
| `sunshine_change` | 日照时长感知类别变化 | sunshine, **perception**, season, time_of_day |
| `precipitation_start` | 降雨开始 | precip_type (rain/snow), intensity, time_of_day |
| `precipitation_stop` | 降雨停止 | time_of_day |
| `season_change` | 季节切换 | season, time_of_day（全局，location=(0,0)） |
| `sunrise` | per-chunk 日出 | time_of_day, daylight_hours |
| `sunset` | per-chunk 日落 | time_of_day, daylight_hours |
| `cold_snap_start` | 寒潮进入 | temperature_offset, time_of_day |
| `cold_snap_stop` | 寒潮退出 | time_of_day |
| `heat_wave_start` | 热浪进入 | temperature_offset, time_of_day |
| `heat_wave_stop` | 热浪退出 | time_of_day |
| `storm_start` | 暴风雨进入 | wind_multiplier, rain_multiplier, time_of_day |
| `storm_stop` | 暴风雨退出 | time_of_day |

### 精确查询 API

需要精确值的模块（UI 面板、温度计、生态模拟）直接调用 API，不依赖事件推送：

```python
# 当前时刻天气（含所有修改器效果）
params = engine.get_weather(cx, cy)           # → WeatherParams

# 任意时刻
params = engine.get_weather(cx, cy, time)

# 感知标签字典
perceptions = engine.get_perceptions(cx, cy)  # → {"temperature": "凉爽", ...}

# 日出日落 + 当前日照强度（含降雨衰减）
sr, ss, daylight, intensity = engine.get_daylight_info(cx, cy)
# intensity 范围 [0, 1]，正弦曲线 × 降雨衰减 × 大气噪声
```

网络层通过 `get_weather` 请求类型开放查询，前端每 0.5s 轮询一次更新 F3 天气面板。

## 极端天气（天气修改器）

低频事件，对天气参数施加临时修改。数据驱动可扩展——添加新类型只需在 `WEATHER_MODIFIERS` 注册表中加一行配置。

### 设计原则

- **各气候带有不同频率**——寒潮只发生在温带/寒带，热浪主要发生在热带/沙漠/草原，暴风雨在热带最频繁
- **效果分两类**：temperature（温度偏移，单位 °C）和 multiplier（风速+降雨倍率）
- **事件驱动**——与降雨相同的 start/stop 配对发布模式
- **可叠加**——多个修改器同时活跃时，temperature 类累加、multiplier 类叠乘

### 三种修改器

| 修改器 | 效果 | 典型强度 | 典型持续 | 发生气候带 |
|---|---|---|---|---|
| 寒潮 (cold_snap) | 温度下降 | -15°C | 3 天 | 草原/温带/亚寒带/极地/高山 |
| 热浪 (heat_wave) | 温度上升 | +15°C | 5 天 | 热带草原/沙漠/草原/温带/高山 |
| 暴风雨 (storm) | 风速+降雨倍增 | ×3.0 | 6 小时 | 全气候带（沙漠极罕见 0.1/年） |

### 游戏影响

- **生存压力**：寒潮/热浪期间，不适应极端温度的个体获得负面标签（"寒冷不适""中暑"），通过标签系统送入 LLM 上下文
- **种植风险**：寒潮可能冻死作物，热浪加速土壤蒸发
- **暴风雨**：高风速影响体感温度和飞行生物，强降雨可能引发局部洪涝
- **基因选择压**：频繁的极端天气事件驱动玩家选择更宽温度适应区间的基因型

## 基因适配

每个天气参数定义生物的基因适应区间。例如：
- 耐寒基因：温度适应下限降到 -20°C
- 耐旱基因：年降雨量适应下限降到 200mm
- 喜光基因：日照时长适应下限提升到 10 小时

生物的实际适应区间由多个基因叠加决定。超出区间的个体产生负面标签（"寒冷不适""脱水"），通过标签系统送入 LLM 上下文。

## 与其他模块的接口

- → **群系生态**：气候档位决定群系分布；连续参数值决定群系内的具体物种构成
- → **生命个体**：天气参数直接输入生理需求计算（温度→体温、湿度→体感舒适、海拔→含氧量→体能消耗）
- → **基因系统**：天气参数区间作为基因适应性的判定基准
- → **玩家行动**：种植条件由天气参数的每日值决定（温度窗口、降雨量、日照时长）
- → **时间系统**：季节调制由世界时钟的 `minute_change` 事件驱动；WeatherEngine 订阅该事件作为唯一 tick 来源
- → **事件系统**：15 种天气事件通过 WorldTree 发布（感知类别变化 + 离散切换），任何下游系统可订阅
- → **表现层**：`get_weather` API 通过 `weather_handler` 响应前端轮询；感知事件转发至 Godot 事件日志
