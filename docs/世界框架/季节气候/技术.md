## 解析算公式

每游戏分钟对每个注册 chunk 从头计算（无快照、无缓存）：

```python
# 温度：四层叠加 + 极端天气修改器偏移
temperature = baseline_temp
            + seasonal_temp_offset(season, day_of_season, seasonal_amp)   # 余弦，±amplitude
            + diurnal_temp_offset(hour, diurnal_amp)                      # 余弦，±amplitude，峰值14:00
            + atmosphere.sample(world_x, world_y, now) * 5.0              # 扰动 ±5°C
            + modifier_temp_offset(now)                                   # 寒潮/热浪 ±15°C

# seasonal_amp 从年均温+年降雨连续推导（_derive_seasonal_amp），非气候带离散取值：
#   base = lerp(28, 2, (T - (-5)) / (35 - (-5)))   # 极地28°C, 赤道2°C
#   rain_bonus = clamp((2000 - R) / 2000, -0.5, 1.0) * 4   # 干旱+4, 海洋调节-2
#   seasonal_amp = clamp(base + rain_bonus, 1, 30)
# 保证气候带交界处温度无跳变（baseline 连续 → seasonal_amp 连续）

# 湿度：四层叠加
humidity = baseline_humidity
         + seasonal_humidity_offset(season, dos, humidity_seasonal_amp, sharpness)  # 余弦或tanh阶梯
         + diurnal_humidity_offset(hour, humidity_diurnal_amp)                      # 反比于温度曲线
         + atmosphere.sample(world_x, world_y, now) * 15.0                          # 扰动 ±15%

# 风速：两层 + 极端天气倍率
wind_speed = (baseline_wind + atmosphere.sample(world_x, world_y, now) * 4.0)  # 扰动 ±4 m/s
           * modifier_wind_multiplier(now)                                       # 暴风雨 ×1.5-4.5

# 日照：天文计算 + 大气扰动（不再使用基线+季节偏移）
# baseline.sunshine 固定为 12.0（天文年均），运行时直接用 daylight_hours() 实时算
day_of_year = (now // GAME_DAY) % 360
sunshine = daylight_hours(day_of_year, latitude)               # 天文日照时长（随纬度+季节变化）
         + atmosphere.sample(world_x, world_y, now) * 1.5      # 云量扰动 ±1.5h

# 降水：事件驱动 + 极端天气倍率
rainfall = rain_schedule.intensity(now) * modifier_rain_multiplier(now)
```

所有结果钳制在物理边界内（温度 [-30, 50]、湿度 [0, 100]、风速 [0, 50]、日照 [0, 24]）。

## 气候生成链

```
纬度噪声 [-1,+1] → 海平面温度 (-5~35°C)
降雨噪声 [-1,+1] → 年降雨量 (50~3500mm)
海拔 (m)
    ↓
气温直减率 (9°C/1000m): 实际温度 = 海平面温度 - 海拔 × 9/1000
    ↓
classify(年均温, 年降雨, 海拔) → ClimateZone (8 档判定)
    ↓
annual_baseline(...) → WeatherParams (六参数基线，存入 ChunkData)
    ↓
WeatherEngine.register_chunk(cx, cy, baseline, climate)
    ↓
运行时解析算 → 发布 per-parameter 事件
```

### 气候档位判定（classify）

纯静态决策树，顺序优先：

1. 海拔 ≥ 2000 → ALPINE（覆盖纬度气候）
2. 温度 < -5 → POLAR_TUNDRA
3. 降雨 < 200 → DESERT
4. 降雨 < 600 且 T > 5 → STEPPE
5. T ≥ 20 → EQUATORIAL_RAINFOREST (R≥1500) / TROPICAL_SAVANNA
6. T ≥ 5 → TEMPERATE_FOREST
7. -5 ≤ T < 5 → SUBARCTIC_TAIGA (R≥400) / POLAR_TUNDRA

## WeatherEngine

核心编排器，模块中唯一持有可变状态的类。

### 生命周期

```python
engine = WeatherEngine(clock, seed=42)        # 构造：订阅 minute_change，注册事件 schema
engine.register_chunk(cx, cy, baseline, climate)  # 注册：每个已加载 chunk
engine.shutdown()                                 # 销毁：取消订阅
```

### 每 tick 流程

```
minute_change 事件 (from GameCalendar)
    │
    ├─ 全局事件检测
    │   └─ season 变化 → 发布 season_change（location=(0,0)）
    │
    └─ per-chunk 循环
        ├─ _compute_params() → WeatherParams
        ├─ 温度感知类别变化 → 发布 temperature_change（含 perception）
        ├─ 湿度感知类别变化 → 发布 humidity_change（含 perception）
        ├─ 风速感知类别变化 → 发布 wind_change（含 perception）
        ├─ 日照时长感知类别变化 → 发布 sunshine_change（含 perception）
        ├─ per-chunk 昼夜切换（用 chunk 纬度算 sunrise/sunset）
        │   ├─ 夜→昼 → 发布 sunrise（含 daylight_hours）
        │   └─ 昼→夜 → 发布 sunset（含 daylight_hours）
        ├─ RainSchedule.pop_due() 状态切换
        │   ├─ 开始下雨 → 发布 precipitation_start（含 precip_type=rain/snow）
        │   └─ 雨停 → 发布 precipitation_stop
        ├─ _replenish_rain() → 裁剪过期 + 补算到 4 场预排
        ├─ ModifierSchedule.pop_due() × N 类型（寒潮/热浪/暴风雨）
        │   ├─ 状态进入 → 发布 {type}_start
        │   └─ 状态退出 → 发布 {type}_stop
        └─ _replenish_modifier() → 裁剪过期 + 补算到 2 场预排
```

事件触发从数值阈值改为感知类别比较：`classify_temperature(params.temperature) != field.last_temp_perception`。类别由 `config.py` 的 `*_PERCEPTION_BOUNDARIES` 定义。

### 线程安全

由 GameEngine 后台单线程驱动，自身不做并发保护。

### 降水类型判定

```python
precip_type = "snow" if temperature <= 0 else "rain"
```

### per-chunk 昼夜切换

每个 chunk 用自己的纬度（`_ChunkWeatherBaseline.latitude`）算当日日出/日落时刻，检测昼夜切换并发布 sunrise/sunset 事件（携带 `daylight_hours = sunset - sunrise`）。不同纬度 chunk 的昼夜时刻独立，极地极昼极夜自然体现。`season_change` 仍为全局事件（location=(0,0)）。

## 季节计算（season.py）

纯函数，无状态。

```python
season_of(day)           # 游戏日 → Season(0=春,1=夏,2=秋,3=冬)，每季 90 天
day_of_season(day)       # 季节内日 [0, 90)
day_of_year(day)         # 年内日 [0, 360)

seasonal_temp_offset(season, dos, amplitude)  # 余弦曲线，夏季中点 +amp，冬季中点 -amp
seasonal_humidity_offset(season, dos, amp, sharpness)
    # sharpness=0: 标准余弦
    # sharpness=2.5 (MONSOON): tanh 阶梯化，旱湿切换突兀
```

## 昼夜计算（diurnal.py）

纯函数，无状态。

```python
diurnal_temp_offset(hour, amplitude)     # 余弦，峰值 14:00(+amp)，谷值 02:00(-amp)
diurnal_humidity_offset(hour, amplitude) # 反比余弦，峰值 14:00 湿度最低
hour_of_game_time(game_time)             # tick → 当日小时 [0, 24)

# 日出/日落（基于太阳赤纬 + 纬度）
sunrise_hour(day_of_year, latitude_deg)  # 日出时刻 [0, 12]
sunset_hour(day_of_year, latitude_deg)   # 日落时刻 [12, 24]
daylight_hours(day_of_year, latitude_deg)  # 当日天文日照时长 = sunset - sunrise [0, 24]
```

日出/日落使用标准天文公式：

- 太阳赤纬 = 23.44° × sin(2π × (day - 45) / 360)
- 半昼长 = arccos(-tan(lat) × tan(decl)) / 15
- 日出 = 12 - 半昼长，日落 = 12 + 半昼长
- 处理极昼（日出=0h）和极夜（日出=12h）边界

`daylight_hours` 同时用于：① sunshine 季节振幅计算（夏至-冬至差的一半）；② sunrise/sunset 事件携带的当日日照时长（供下游种植/生理系统）。

per-chunk 昼夜切换用 chunk 自己的纬度（来自 `_CLIMATE_LATITUDE` 气候带映射），不再使用全局参考纬度。

## 大气扰动场（atmosphere.py）

```python
class AtmosphereField:
    def __init__(self, seed, resolution=2000m, drift_rate=1e-5)
    def sample(world_x, world_y, game_time) -> float   # 返回 [-1, 1]
    def wind_vector(game_time) -> (float, float)       # 单位向量
```

- 主噪声场：2D Perlin，采样间距 2km
- 气团漂移：采样坐标 = 空间坐标/resolution + wind_vector × drift_rate × game_time
- 风向：由第二个 Perlin（seed+1）驱动，tick × 1e-7 的极慢速推进 → 季节性风向变化
- 相邻 chunk 坐标接近 → 扰动值空间连续，O(1) 无需邻居查询

## 降水调度（rain_events.py）

```python
class RainSchedule:
    def __init__(rng, annual_rainfall, mean_intensity, mean_duration_h, ramp_up, ramp_down)
    def push(event)           # 追加事件（start_tick 严格递增）
    def seed_current(now)     # 初始化降雨状态基线
    def pop_due(now) -> bool  # 检测状态切换
    def intensity(now) -> float       # 当前强度（事件外为 0）
    def is_raining(now) -> bool       # 是否处于事件区间
    def generate_next(earliest) -> RainEvent  # 确定性随机生成
    def prune_before(tick)            # 裁剪已结束事件
    def needs_replenish(threshold)    # 队列是否需补算
```

### 强度曲线

```
intensity_at(event, now, ramp_up=0.2, ramp_down=0.2):
    progress = elapsed / duration
    if progress < ramp_up:      线性爬升 0 → peak
    if progress < 1 - ramp_down: 峰值平台
    else:                        线性衰减 peak → 0
```

### 频率推算

```python
rain_hours = annual_rainfall / mean_intensity
n_events = max(1, rain_hours / mean_duration)
mean_interval_hours = (year_hours - rain_hours) / n_events
```

### 随机化

每个参数在 0.5x-1.5x 均值范围内随机抖动，使用 chunk 坐标派生的 `random.Random` 实例保证确定性。

## 极端天气修改器（weather_modifier.py）

数据驱动的低频天气修改器系统，对天气参数施加临时修改（温度偏移、风速/降雨倍率）。添加新修改器只需在 `WEATHER_MODIFIERS` 注册表中加一行。

### 架构

```python
@dataclass(slots=True)
class ModifierConfig:
    type_name: str                              # "cold_snap" / "heat_wave" / "storm"
    rates: dict[ClimateZone, float]             # 各气候带年均事件数，0=永不发生
    mean_duration: int                          # 典型持续时长（tick），实际 0.5x-1.5x 随机化
    base_intensity: float                       # 基准强度
    effect: str                                 # "temperature" | "multiplier"
    start_schema: dict[str, type]               # start 事件的 required 字段
```

### 当前注册表

| 修改器        | 效果类别    | 基准强度 | 典型持续 | 主要气候带                              |
| ------------- | ----------- | -------- | -------- | --------------------------------------- |
| `cold_snap` | temperature | -15°C   | 3 天     | 亚寒带(2.0)、极地(2.0)、高山(1.5)       |
| `heat_wave` | temperature | +15°C   | 5 天     | 沙漠(2.0)、草原(1.5)、热带草原(1.0)     |
| `storm`     | multiplier  | ×3.0    | 6 小时   | 热带草原(2.0)、热带雨林(1.5)、温带(1.0) |

### ModifierSchedule

```python
class ModifierSchedule:
    def __init__(rng, config, climate)        # 从 rates[climate] 推算 mean_interval
    def push(event)                           # 追加事件（start_tick 严格递增）
    def seed_current(now)                     # 初始化活跃状态基线
    def pop_due(now) -> bool                  # 检测状态切换
    def is_active(now) -> bool                # 是否处于事件区间
    def temp_offset(now) -> float             # 温度偏移（仅 temperature 类）
    def wind_rain_multiplier(now) -> float    # 倍率（仅 multiplier 类）
    def generate_next(earliest) -> ModifierEvent  # 确定性随机生成
    def prune_before(tick) / needs_replenish(threshold)
```

队列管理与 `RainSchedule` 同模式：预排 `MODIFIER_FORECAST_DEPTH=2` 场，低于 `MODIFIER_REPLENISH_THRESHOLD=1` 时补算。各参数在 0.5x-1.5x 均值范围内随机化。

### 效果叠加

- 多个 temperature 类修改器同时活跃时，偏移量**累加**（如寒潮 + 热浪同时发生则相互抵消）
- 多个 multiplier 类修改器同时活跃时，倍率**叠乘**
- 修改器效果在 `_compute_params()` 中施加，位于季节/昼夜/扰动之后、钳制之前

## 事件注册（events.py）

模块导入时即向 WorldTree 单例注册天气事件 schema。WeatherEngine 构造时也在注入的 WorldTree 实例上注册（测试隔离）。

感知类事件的 schema 增加 `perception: str` 字段：

- `temperature_change`: {temperature: float, **perception: str**, season: int, time_of_day: int}
- `humidity_change`: {humidity: float, **perception: str**, time_of_day: int}
- `wind_change`: {wind_speed: float, **perception: str**, wind_dir_x: float, wind_dir_y: float, time_of_day: int}
- `sunshine_change`: {sunshine: float, **perception: str**, season: int, time_of_day: int}
- `precipitation_start`: {precip_type: str, intensity: float, time_of_day: int}
- `precipitation_stop`: {time_of_day: int}
- `season_change`: {season: int, time_of_day: int}（全局，location=(0,0)）
- `sunrise` / `sunset`: {time_of_day: int, daylight_hours: float}（per-chunk）

极端天气修改器事件由 `WEATHER_MODIFIERS` 注册表自动生成。

## 查询 API

WeatherEngine 提供四个公开查询方法，供需要精确值的模块（UI 面板、生态模拟）直接调用。
**查询仅支持当前或过去时刻，未来时刻抛 `ValueError`**；降雨/修改器效果仅对调度保留窗口内的时刻精确（更久远的历史时刻降雨事件已被修剪，rainfall 返回 0）：

```python
# 当前/过去时刻的完整天气参数（含所有修改器效果）
def get_weather(cx, cy, time=None) -> WeatherParams | None

# 感知标签字典 {"temperature": "凉爽", "humidity": "舒适", ...}
def get_perceptions(cx, cy, time=None) -> dict[str, str] | None

# 日出日落 + 日照强度（降雨衰减已含暴风雨修改器）
def get_daylight_info(cx, cy, time=None, rainfall=0.0) -> (sr, ss, daylight, intensity) | None
# intensity ∈ [0, 1], 正弦曲线 × rain 衰减 × 大气噪声

# 当前时刻完整报告（网络 handler 专用）：天文与噪声只算一次，降雨衰减自动含修改器
def get_weather_report(cx, cy) -> (WeatherParams, sr, ss, daylight, intensity) | None
```

网络层通过 `weather_handler.py` 的 `get_weather` 请求类型对外暴露：逐坐标校验（畸形坐标跳过不毁整批）、单请求上限 `MAX_WEATHER_QUERY_CHUNKS`；所有感知标签按**四舍五入后的显示值**分类（数值与标签一致），response 用 `sunshine`（含扰动的日照时长）字段。前端 F3 面板每 0.5s 轮询一次更新天气分区（实时值）；气候分区显示 chunk 年均基线。

## 参数存储

- 每个 chunk 的"年均基线"（WeatherParams）存储在 `ChunkData.annual_baseline`，chunk 生成时由 `annual_baseline()` 确定。日照基线固定为 12.0（天文年均），季节变化由运行时 `daylight_hours()` 实时计算
- WeatherEngine 注册时从基线 + 气候模板派生 `_ChunkWeatherBaseline`（含温度/湿度/风速/降雨/日照基线 + 季节振幅（从年均温+降雨连续推导）/昼夜振幅 + SeasonalityMode + 纬度）
- 实时天气值不持久化——每 tick 解析算

## 整合点

- **GameEngine** (`game.py`)：创建 WeatherEngine，遍历已加载 chunk 调用 `register_chunk()`，停止时调用 `shutdown()`；注册 weather_handler 至 MessageDispatcher
- **GameCalendar** (`time/calendar.py`)：发布 `minute_change` 事件（含 `game_time`, `day`, `hour`, `minute`），是天气引擎的唯一 tick 来源
- **WorldTree** (`world_tree/`)：事件总线，天气引擎在此注册 schema 并发布 15 种事件
- **ChunkData** (`space/chunk.py`)：提供 `annual_baseline` 和 `climate_zone` 作为天气注册输入
- **weather_handler** (`net/handlers/weather_handler.py`)：处理前端 `get_weather` 请求，调用 `get_weather()` + `get_daylight_info()`，通过 i18n 翻译感知标签

## 下游消费

当前已实现的消费者：

- **EventBridge**：通配符订阅全部天气事件，转发至 Godot 前端事件日志
- **F3 天气面板**：通过 `get_weather` API 每 0.5s 轮询，显示温度/湿度/风速/光照/日出日落
