# 事件系统 — 设计

事件系统是连接所有模块的**总线**。

就像硬件总线——CPU、内存、外设不直接通信，而是通过总线交换数据。在 Ascend 中，气候系统不直接告诉 NPC 下雨了，生态变化不直接通知聚落，NPC 之间的互动也不走私有的通信管道。所有模块都向事件总线发布事件，所有模块都从事件总线订阅自己关心的事件。

世界中的一切变化——从一场雨到一个文明兴衰——都作为事件在总线上流动。

## 事件的定义

**状态的变化**即为事件。持续状态（"正在下雨"）不是事件，状态改变的瞬间（"开始下雨""雨停了"）是事件。

事件由各自的系统生成，发布到总线上：
- 该不该下雨 → 气候系统负责
- 该不该吃饭 → 行为系统负责
- 玩家建造 → 玩家行动系统负责

事件系统本身**不生成事件**，只负责总线本身的功能：记录、路由、索引、查询。各模块之间不直接耦合——模块 A 不知道也不关心模块 B 是否在监听，只发布事件到总线上。

## 事件结构

每个事件包含：
- **发起方**：谁/什么触发了这个事件（系统、NPC、玩家）
- **受影响方**：谁/什么受到了影响（可以是实体集合、区域、整个世界）
- **事件数据**：发生了什么的具体内容
- **时间戳**：世界时间
- **位置**：事件发生的地点
- **因果关系**：引用其他事件（caused_by, observes, informed_by 等）

## 按发起方分类

### 系统事件

非人格系统发起。气候（下雨、暴风）、生态（开花、种群迁移）、物理法则范畴内的变化。

特征：无主观意图，纯因果链驱动。

### NPC 事件

NPC 发起。包括：
- **行动事件**：做什么（吃饭、建造、战斗、交易）
- **言语事件**：说什么（对话、承诺、传闻）
- **观测事件**：注意到什么（感知层自动生成，详见下文）
- **情感事件**：对某事产生情感反应
- **决策事件**：做出选择（影响后续行动链）

### 玩家事件

玩家发起。与 NPC 事件结构相同，但发起方为玩家。玩家的所有行动也走事件系统，不做特殊对待——玩家不是"事件系统之外的观察者"，而是事件的参与者。

## 观测的分层

观测事件不是"NPC 主观决定去注意"的结果。它有物理前提和认知加工两层：

### 感知层（自动，不受主观控制）

由物理条件判定：
- 是否在感知范围内（距离）？
- 是否有遮蔽（室内/室外、障碍物）？
- 感官是否正常（没在睡觉、没失明失聪）？

条件满足 → 自动生成观测事件。这是"客观上感知到了"，不消耗注意力资源，不需要 LLM。

### 注意层（主观，受行为系统控制）

对已观测到的事件，NPC 的行为系统决定关注程度：
- 只是知道（"昨天下雨了"）→ 低关注，浅记忆
- 影响自身目标（"下雨会摧毁我的农田→需要抢收"）→ 高关注，深记忆

关注程度影响记忆深度和后续决策。只有进入注意层、需要影响决策时，才唤醒 LLM。

## 事件图

事件之间不是简单的线性链条，而是**有向图**。关系类型包括：

| 关系 | 含义 | 示例 |
|------|------|------|
| `caused_by` | A 导致了 B | 暴雨 → 洪水冲毁粮仓 |
| `observes` | 观测事件引用被观测事件 | NPC 注意到下雨 → 引用下雨事件 |
| `co_participant` | 多方共同参与 | 婚礼中双方都是发起方 |
| `informed_by` | 从他人处获知 | 从邻居那里听说某件事 |

## 共享事件

当一个事件有多个共同参与者时（如结婚、合作、冲突），每个参与者都将其记录在自己的时间线上。不是复制——是同一个事件，所有参与方的时间线都引用到它。

## 事件与快进

快进期间，事件系统是"故事生成器"：

1. 生态/气候/社会系统持续产出系统事件
2. NPC 在行为系统驱动下响应事件，产生新事件
3. 因果链自然展开：事件 A → 影响 NPC X 的目标 → NPC X 发起事件 B → ...
4. LLM 在因果分叉点介入——当 NPC 面临需要智能判断的抉择时
5. 事件图持续增长，快进结束后玩家可按时间线回顾重要事件

不是一次性跳过几十年然后生成摘要，而是事件链逐个推演。

## 事件的遗忘

NPC 的记忆有限。事件随时间衰减：

- 日常琐事（天上下雨、今天吃了什么）→ 短时间内淡出活跃记忆
- 情感重大事件（结婚、亲人死亡）→ 长期记住
- 快进后，只有"重大事件"保留在活跃记忆中，其余归档

重要性判定由注意层关注程度 + 事件本身的客观影响范围共同决定。

## 总线上的各模块

所有模块通过事件总线连接，发布和订阅事件：

- → **时间系统**：时间推进是总线的主时钟，每个事件带上时间戳；快进期间时钟加速，事件流随之加速推演
- → **气候系统**：向总线发布天气变化事件；不关心谁在听，只负责发布
- → **生态系统**：向总线发布生态变化事件（种群波动、花期）
- → **心智系统**：订阅与 NPC 相关的事件（感知判定通过后自动生成观测事件）；注意层决定关注程度；LLM 在因果分叉点被事件唤醒
- → **群体社会**：订阅聚落范围内的事件；个人事件可升级为聚落事件，聚落事件可升级为世界事件，通过 `caused_by` 链式发布到总线上
- → **玩家行动**：玩家的所有行动作为玩家事件发布到总线，同等流转
- → **表现层**：订阅总线，重要事件触发视觉/音效/UI 通知
