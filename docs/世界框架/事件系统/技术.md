# 事件系统 — 技术

## 事件数据结构

```
Event {
  id: UUID,
  timestamp: WorldTime,
  location: (chunk_x, chunk_y, tile_x?, tile_y?) | region_id,

  initiator_type: "system" | "npc" | "player",
  initiator_id: string,          // 系统名 / NPC ID / player
  affected: [AffectedParty],     // {entity_id, role: "subject"|"witness"|"recipient"}

  event_type: string,            // "weather_change" "npc_action" "observation" ...
  data: JSON,                    // 事件类型特定的内容

  // 事件图关系
  caused_by: [EventID],          // 因果上游
  observes: EventID?,            // 观测引用（仅观测事件）
  co_participants: [EntityID],   // 共享参与方
}
```

## 事件类型注册

每个产生事件的系统注册自己的事件类型 schema：
- 气候系统注册：`weather_change`（参数变更）、`weather_extreme`（极端天气）
- 行为系统注册：`npc_action`、`npc_speech`、`npc_decision`
- 生态系统注册：`population_change`、`bloom`、`migration`
- 感知系统注册：`observation`

事件系统本身不校验数据内容，只校验结构完整性（必填字段、引用有效性）。

## 观测事件的感知判定

感知判定是轻量的物理计算，不涉及 LLM：

```
can_perceive(observer, event):
  if distance(observer.position, event.location) > observer.perception_range:
    return false
  if is_occluded(observer.position, event.location, tile_map):
    return false
  if observer.state in {sleeping, unconscious, blind, deaf}:
    return false
  return true
```

通过判定 → 生成 observation 事件，引用原事件 `observes`。

## 事件存储

### 世界事件日志

全局有序事件流。按时间戳存储，每条事件一条记录。仅追加，不修改。

### NPC 时间线索引

每个 NPC 维护一个事件引用列表，指向世界事件日志。是索引而非副本——共享事件只有一个存储位置，所有参与者通过索引指向它。

索引结构：`{event_id, relevance_score, memory_depth}`

- `relevance_score`：对 NPC 的重要性，由注意层计算
- `memory_depth`：随世界时间衰减，深度低于阈值的事件从活跃索引中移除（归档而非删除）

## 事件图查询

支持沿关系边遍历：
- `get_causal_chain(event_id, depth)` → 沿 `caused_by` 向上追溯
- `get_consequences(event_id)` → 反向：哪些事件引用了此事件为因
- `get_observers(physical_event_id)` → 哪些 NPC 观测到了此事件
- `get_timeline(npc_id, time_range, min_relevance?)` → NPC 时间线，可选过滤低重要性事件

## 事件升级

个人事件 → 聚落事件 → 世界事件的升级判定不在事件系统内部，而由相应的上层系统（群体社会、心智系统）判定后，生成新的事件，通过 `caused_by` 引用原始事件。

事件系统只提供查询支持——"这个个人事件当前影响了多少人？"——供上层做升级判定。

## 快进推演循环

```
while world_time < target_time:
    step_size = determine_step_size()  // 根据当前局势动态调整
    events = []
    for system in active_systems:
        events += system.tick(step_size)  // 各系统产出事件
    for event in events:
        affected_npcs = get_affected_npcs(event)
        for npc in affected_npcs:
            if npc.needs_llm_judgment(event):   // 因果分叉点检测
                decision = llm_judge(npc, event, context)
                events += behavior.execute(npc, decision)
        store_events(events)
        update_world_state(events)
    world_time += step_size
```

各系统的 `tick()` 返回事件列表，事件系统负责存储和路由。本身不生成事件。

## 记忆衰减

```
memory_depth(t) = initial_depth × e^(-λ × t)
```

- `initial_depth` 由注意层关注程度决定
- `λ`（衰减速率）由事件类型决定：日常琐事 λ 大，情感重大事件 λ 小
- 低于阈值的记录从活跃索引移除，进入归档区
- 归档数据保留但不参与 LLM prompt 上下文构建
