# 地图生成 — 设计

## 双层地图结构

```
大地图（选点/远行）          详细地图（活动/建造/社交）
─────────────────────      ─────────────────────────
          ┌───┬───┬───┐
          │   │   │   │    每个格子展开 →
          ├───┼───┼───┤    固定尺寸的 tile 地图
          │   │ ■ │   │    200×200 tile
          ├───┼───┼───┤
          │   │   │   │
          └───┴───┴───┘
```

- **大地图**：无限延伸的网格，每个格子是一个分块。俯瞰视角，用于选点、远行规划、了解区域分布。分块的核心作用是**大地图坐标定位 + 群系区分**——每个分块有独立的群系标签，格子的密度决定了群系地图的精细度
- **详细地图**：玩家脚下的无缝连续 tile 地图。没有分块边界——走到哪加载到哪，分块只是大地图层的概念，不影响详细地图的连续性

### 两层生成

地图分两层生成，层 1 一次生成整个大陆宏观场，层 2 按玩家所在区域按需展开为详细 tile：

- **层 1 宏观场**（100m/px，1000×600 格）：海拔、气候（温度+降雨）、水力侵蚀后的 DEM、河流网络、湖泊盆地。一次生成约 12s，存于 `ContinentData`，启动时由 `GameEngine` 主动触发
- **层 2 详细 tile**（1m/px，200×200/chunk）：从层 1 双线性采样宏观海拔，叠加高频细节噪声，按海拔带分类地形，再叠加河流河道与湖泊水面。按 chunk 请求时现算，可并行

## 水文

侵蚀与河流是地图的骨架，决定地形起伏与水体分布。

### 侵蚀

降雨驱动的水力侵蚀（12 轮迭代）：水从高处流向低处，沿途侵蚀地表、下游沉积。侵蚀公式正比于流量平方根与坡度，质量守恒。结果是雕刻过的 DEM——山脉被削出河谷，平原堆积沃土。

### 河流

河流网络用**三级方向场追踪**：从流量累积的源头出发，沿混合方向场积分到海。山地跟随海拔梯度（山谷自然弯曲），平原跟随流量场（指向下游主流），分水岭与兜底靠 Dijkstra 到海代价场。追踪后做 Chaikin 平滑，产出连续弯曲坐标的河流路径，消除格子线。

河流有 Strahler 分级（主流深、支流浅），宽度随流量对数增长。支流在 2 格内碰到主流即汇入，形成树状网络。

### 湖泊

湖泊是侵蚀后留下的洼地——填洼算法检测出"填平后升高超 1m"的区域，每个连通分量是一个湖盆。湖面高度 = 盆地最低溢出口（水满则溢）。湖泊大小取决于洼地面积，是侵蚀的被动副产物而非主动放置。

## Z 轴

为支持洞穴层（提供特殊基因生物的栖息地），地图引入 Z 轴，但拆成两个正交概念避免 3D 复杂度：

### 渲染 Z（连续高度）

每个 tile 存连续高度值（float），供 isometric 渲染抬升 tile 顶面展现高差；同时存局部梯度值（float），供渲染选择平坦/斜坡 tile 变体。这是纯视觉数据，事件/寻路/碰撞不碰——"tile 高 23.5m 还是 24m"对游戏逻辑无意义。

### 逻辑 Z（离散层 id）

层 id 枚举：`0=地表，-1=浅洞，-2=深洞`。每层内部是纯 2D 平面，寻路/碰撞/视野/势力范围全保持 2D 算法。

- 实体和事件带 `layer_id` 字段（默认 0=地表），空间索引按 `(layer_id, chunk_x, chunk_y)` 分桶——不同层天然隔离
- 跨层交互用两个 2D 事件联动（地表塌方 ↔ 洞穴顶部塌落），而非 3D 事件
- 洞穴生物挂在 `layer_id=-1`，与地表种群隔离，杂交/迁徙需显式跨层动作——正好符合"特殊基因不易混入"的设计意图

洞穴层的 chunk 数据结构待定，当前只实现了地表层（layer_id=0）。

## 分块

### 定位

分块 ≠ 加载单元。分块是大地图层的概念——给世界贴上一层坐标网格，每个格子标记一种群系。详细地图是完整连续的，玩家行走时感受不到分块边界。

分块的三重作用：
- **坐标定位**：大地图上选点、远行路径规划、标记地点
- **群系区分**：每个分块一个群系类型，格子的密度决定了群系地图的精细度
- **并行单元**：不同分块的模拟可分配到不同线程

### 加载

只加载玩家视野范围内的 tile——无论跨了多少分块边界。视野外的 tile 不占内存。

## 大地图

### 分块信息

大地图上每个分块格子显示：
- 群系类型（森林/草原/沙漠/沼泽/山地/水域等）
- 已知标记（聚落位置、遗址、资源点、玩家建造的设施）
- 通行属性（能否穿越、移动速度）

### 大地图与详细地图的切换

- **放大**：从详细地图缩放到大地图——俯瞰当前分块及周边
- **缩小**：从大地图进入某个已探索分块的详细地图——加载该分块 tile
- 大地图上看不到详细地图内部的 NPC、建筑细节——只知道"这里有个聚落""这里有座塔"

### 大地图上的活动

- 选点：查看群系分布，决定去哪里探索/采集/建立新据点
- 快速旅行（长途）：选中目标分块 → 计算路径和时间 → 触发快速旅行模式（时间系统定义，几周到几个月）
- 标记：手动在大地图上做标记——"这里有稀有基因生物""前方危险"

### 迷雾

未探索的分块在大地图上为迷雾状态。探索方式：
- 亲身走到相邻分块（详细地图步行跨过分块边界）
- NPC 告知（对话中 NPC 描述远方的群系/聚落，自动揭露对应分块的基础信息）
- 远行途径时自动揭露沿途分块

## 群系生成

每个分块在首次生成时确定群系类型。群系由噪声函数分配，相邻分块的群系有自然过渡——不会出现"热带雨林隔壁是冻土"。

群系决定分块内的：
- 地形构成（水体比例、山地分布、平原面积）
- 生物种类分布（野生生物、怪物）
- 资源类型（矿产、可采集植物）
- 环境参数（温度、湿度基线）

群系详细参数由群系生态模块定义。

## 特殊分块

- **长生种聚落分块**：稀有，生成时标记为特殊群系，建筑和 NPC 由群体社会模块管理
- **遗址分块**：内含古代设施/稀有基因/可研究物品
- **玩家建造的分块**：玩家在空分块上从零建造据点。分块内的人工建筑和地形改造持久保留

## 与其他模块的接口

- → **群系生态**：分块生成时，群系类型决定该分块的生物、资源、环境参数
- → **时间系统**：大地图快速旅行触发长途移动时间跳转；分块加载/卸载与时间模式无关
- → **群体社会**：聚落位置注册到大地图分块索引；分块的所有权、势力范围在大地图上可见
- → **玩家行动**：建造只在详细地图内发生；采集和探索依赖分块的群系资源分布
- → **表现层**：大地图用抽象色块+图标渲染；详细地图用 tile 渲染
