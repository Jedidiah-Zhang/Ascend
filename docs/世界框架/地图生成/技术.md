# 地图生成 — 技术

## 分块存储

- 每个分块以坐标 (cx, cy) 为唯一标识
- 分块数据分两层：大地图层（群系、标记、通行性）和详细层（tile 网格 200×200）
- 未加载的分块只保留大地图层数据（轻量），详细层按需生成/加载
- 玩家改造过的分块（建筑、地形修改）持久化存储详细层

## 实现模块结构

```
ascend/world/
├── noise.py       Perlin 噪声（种子化、八度叠加、线程安全）
├── climate.py     ClimateZone 枚举 + 六参数基线噪声→参数映射
├── biome.py       BiomeType 枚举 + BiomeTemplate + 群系分配
├── chunk.py       ChunkData 数据结构（大地图层 + 详细层）
└── generator.py   WorldGenerator 协调器（串行/并行生成）
```

### 各模块线程安全设计

| 模块 | 策略 |
|------|------|
| `noise.py` | 每个线程创建独立 PerlinNoise 实例（种子相同则结果相同），无共享状态 |
| `climate.py` | 纯函数，从噪声值计算参数基线，无状态 |
| `biome.py` | 纯函数，从气候参数映射群系，无状态 |
| `chunk.py` | ChunkData 是值对象，不可变的大地图层 + 可变的详细层（创建后归属调用线程） |
| `generator.py` | 协调分块生成，使用 `concurrent.futures` 并行调度，可注入线程池 |

## 生成流程（线程视角）

```
主线程:
  1. 创建 WorldGenerator(seed) → 初始化噪声、群系模板
  2. 计算目标分块列表 [(cx1,cy1), (cx2,cy2), ...]
  3. 并行生成:
     ┌─ Thread-1: noise→climate→biome→chunk(cx1,cy1)
     ├─ Thread-2: noise→climate→biome→chunk(cx2,cy2)
     └─ Thread-N: noise→climate→biome→chunk(cxN,cyN)
  4. 收集结果，相邻分块边界匹配
```

### 边界匹配策略

- 分块群系由全局噪声函数在 (cx, cy) 处采样决定，相邻分块噪声值连续
- 气候参数基线由噪声连续映射，相邻分块自然渐变
- 详细 tile 层生成时：若相邻分块已生成，读取边缘参数做约束；若未生成，用噪声外推边缘值

## 加载窗口

- 以玩家所在分块为中心，加载 3×3 = 9 个分块的详细地图
- 玩家跨过分块边界时，移出窗口的分块卸载详细层（保留大地图层），新进入窗口的分块按需生成/加载详细层
- 大地图视图只读大地图层，详细地图视图读详细层

## 噪声生成

- 分块的年均基线使用多层 Perlin/Simplex 噪声叠加，确保相邻分块渐变
- 不同参数使用不同频率和种子偏移，避免温度和降雨量完全正相关
- PerlinNoise 实例化时从 world_seed 派生子种子，保证确定性

## 并行

- 不同分块独立性强，适合多线程——分块生成、NPC 模拟、生态更新可分配到不同线程
- 视口外分块的模拟在后台线程运行，不阻塞主线程的渲染和玩家交互
- 使用 `concurrent.futures.ThreadPoolExecutor`，可配置线程数
