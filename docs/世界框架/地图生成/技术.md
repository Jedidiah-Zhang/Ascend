# 地图生成 — 技术

## 分块存储

- 每个分块以坐标 (cx, cy) 为唯一标识
- 分块数据分两层：大地图层（群系、标记、通行性）和详细层（tile 网格 200×200）
- 未加载的分块只保留大地图层数据（轻量），详细层按需生成/加载
- 玩家改造过的分块（建筑、地形修改）持久化存储详细层

## 加载窗口

- 以玩家所在分块为中心，加载 3×3 = 9 个分块的详细地图
- 玩家跨过分块边界时，移出窗口的分块卸载详细层（保留大地图层），新进入窗口的分块按需生成/加载详细层
- 大地图视图只读大地图层，详细地图视图读详细层

## 边界无缝

- 分块边缘的 tile 生成时引用相邻分块的群系参数，确保地形、植被在边界处自然连续
- 相邻分块尚未生成时，以噪声坐标外推边缘值，相邻分块生成时匹配该值

## 生成流程

1. 确定分块群系（噪声函数 + 相邻群系约束）
2. 生成大地图层（群系、地形标记、资源标记）
3. 玩家进入详细地图时，按群系参数生成 200×200 tile 网格
4. 首次生成后持久化，后续加载直接读取

## 并行

- 不同分块独立性强，适合多线程——分块生成、NPC 模拟、生态更新可分配到不同线程
- 视口外分块的模拟在后台线程运行，不阻塞主线程的渲染和玩家交互
