# 地图生成 — 技术

## 双层架构

世界地图分两层生成，层 1 是低分辨率宏观场，层 2 是玩家活动粒度的详细 tile。

```
层1 宏观场（100m/px，1000×600）          层2 详细 tile（1m/px，200×200/chunk）
─────────────────────────────────       ─────────────────────────────────────
  ContinentGenerator.generate()            TileGenerator.generate_chunk(cx,cy)
  ├─ 海拔（Perlin + 分位数校准）             ├─ 从层1双线性采样宏观海拔
  ├─ 气候（纬度温度 + 降雨 + 雨影）         ├─ 叠加高频细节噪声(±50m)
  ├─ 侵蚀（12轮，降雨驱动，C加速）          ├─ 按海拔带分类 TerrainType
  │   → DEM + D8流向 + 流量累积 + 沉积       ├─ 叠加河流(render_river_chunk)
  ├─ 湖泊盆地（填洼差>1m的洼地）            └─ 叠加湖泊(render_lake_chunk)
  ├─ 河流网络（三级方向场RK4积分）
  └─ 河流宽度场（log(流量)）
```

层 1 一次生成整个大陆（约 12s），存于 `ContinentData`；层 2 按 chunk 请求时从层 1 采样现算，可并行。

## 层 1 宏观场生成（ContinentGenerator）

`ContinentGenerator(seed).generate()` 产出 `ContinentData`，管线顺序：

1. **海拔 + 陆地掩码**：双层 Perlin（低频大陆轮廓 0.7 + 高频细节 0.3 + 中心倾向），分位数校准到 `land_ratio=0.55`，海平面 = `(mixed - sea_level) × 4000`
2. **海拔校准**：拉伸 P90-P99 段保证陆地 P99 ≥ 2500m（高山存在）
3. **气候**：纬度噪声 → 海平面温度；Perlin → 降雨；雨影因子（4 风向滑动窗口累加上坡量）；气温直减率 `actual_temp = sea_temp - altitude × LAPSE_RATE/1000`
4. **降雨/温度交叉校准**：保证沙漠（<200mm）、热带（≥20°C）、极地（<-5°C）覆盖
5. **侵蚀**（`erode()`，C 加速）：12 轮迭代，每轮 填洼→D8→累积→侵蚀步。侵蚀公式 `K × flow^0.5 × slope`，质量守恒（上游侵蚀、下游沉积）。自适应收敛 `max_delta < tolerance` 提前退出
6. **水文提取**：从侵蚀结果提取河流树（D8 拓扑 + Strahler 分级）、湖泊盆地（填洼差>1m 的连通分量）、河流网络（见下节）
7. **河流宽度场**：`width = min + (max-min) × log(1+ratio×20)/log(21)`，基于 log(流量)

### 河流网络生成（streamlines.py）

`build_river_network()` 用**三级方向场 RK4 积分**追踪自然弯曲河流：

1. **Dijkstra 代价场** `dist[]`：从所有海洋格反向 Dijkstra，`cost(q) = max(0, dem[q]) + 1 - 1.5·log(1+flow_acc[q])`。高海拔贵、高流量便宜。作为全局可达性约束
2. **源头检测**：`flow_acc >= threshold` 且无更高 acc 上游流入的格子，按流量降序
3. **三级方向场 RK4 积分**（每条河从源头追踪到海）：
   - **a. dem 平滑梯度**（高斯 σ=1.5）：跟随山谷自然弯曲。仅当与 dist 下降同向时启用，否则视为跨分水岭
   - **b. flow_acc 平滑场 +梯度**：平原区指向下游高流量主流。须与 dist 下降同向（防局部极大绕圈打转）
   - **c. dist 场**：纯几何兜底，保证到海
4. **Chaikin 切角平滑** 2 轮：消除格子线
5. **汇流合并**：流线经过已有河流 2 格范围内 → 截断并汇入（前段 <15 步碰到则丢弃，消除平行短支流）。截断时追加末端点至主流格子中心，避免间隙
6. **Strahler 分级**：基于 parent_indices 拓扑从叶向根传播

设计取舍：纯 dem 梯度会跨分水岭、纯 dist 平原退化为平行直线、纯 flow_acc 局部极大处打转。三级混合取长补短——山地 dem 弯曲、平原 flow_acc 弯曲、分水岭/兜底靠 dist。

### 湖泊盆地提取（hydrology.py）

`extract_lake_basins()`：被动检测侵蚀后 DEM 留下的洼地。
- 洼地像素 = `filled_dem[i] - dem[i] > 1.0`（填洼后升高超 1m）
- BFS 找 8 连通分量，过滤 <5 像素
- 湖面高程 = 盆地边界像素的 `filled_dem` 最低值（溢出口高度）
- 面积 = 像素数 × 0.01 km²（100m/px）

## 层 2 详细 tile 生成（TileGenerator）

`TileGenerator(seed, continent).generate_chunk_for(chunk)` 返回 `TileGrid`：

1. **宏观海拔采样**：`sample_altitude_bilinear()` 从层 1 双线性插值
2. **细节噪声**：±50m，波长 200m，4 octaves（一次 C 调用算 40K tile）
3. **tile 级群系隶属度**：从层1采样 tile 级气候属性（温度/降雨/海拔/moisture），`biome_membership()` 算对各群系的隶属度
4. **TerrainBias 混合**：`bias = Σ weight_i × bias_i`，数值字段加权平均
5. **地形分类**：`_classify(elev, temp, bias)` 按偏移后海拔带 → `TerrainType`（DEEP_WATER/SHALLOW_WATER/SAND/FERTILE_SOIL/GRASSLAND/ROCK/STEEP_SLOPE/MOUNTAIN_PEAK）。积雪暂时禁用，等天气系统
6. **叠加河流**：`render_river_chunk()` 从层 1 river_network 取穿过的流线，沿点画河道圆（深水→浅水→沃土过渡），插值填充间隙
7. **叠加湖泊**：`render_lake_chunk()` 宏观海拔 < surface_elev → 水体；大湖中央 → DEEP_WATER；湖面以上 0-2m → MARSH 湿地

群系通过 TerrainBias 偏移地形分类阈值，不同群系产出不同地形分布。chunk 边界因隶属度混合而连续（实测 0 跳变）。

## TileGrid 存储（tile_grid.py）

```
TileGrid
├── _data: array('H')       200×200 地形类型（TerrainType 枚举）  80KB
└── _elevation: array('f')  200×200 tile 级连续高度 (m)            160KB
```

高度场供 2.5D 渲染抬升 tile 顶面，及游戏逻辑的精确高度查询。`to_list()` / `to_elevation_list()` 导出给 Godot。

## Z 轴设计

Z 轴拆为两个正交概念，避免事件/寻路/碰撞进入 3D 复杂度：

### 渲染 Z（连续高度）

`TileGrid._elevation` 的 float 值，只给 Godot 抬高顶面用。事件系统、寻路、碰撞不碰它——"这个 tile 高 23.5m 还是 24m"对游戏逻辑无意义。

### 逻辑 Z（离散 layer_id）

层 id 枚举：`0=地表，-1=浅洞，-2=深洞`。每层内部是纯 2D XY 平面。

- `Entity.layer_id: int = 0` — 实体所在 Z 层
- `Event.layer_id: int = 0` — 事件所在 Z 层
- `Event.location` 保持层内平面坐标 `(chunk_x, chunk_y, tile_x?, tile_y?)`，layer_id 作为正交字段
- 空间索引 key = `(layer_id, chunk_x, chunk_y)` 3 元组——不同层落入不同桶，天然隔离

**跨层交互**不做成 3D 事件，而是两个 2D 事件联动：地表塌方 → layer=0 触发"塌方"，同时 layer=-1 同 (x,y) 生成"顶部塌落"事件。寻路、碰撞、视野、势力范围全保持 2D 算法。洞穴生物的"特殊基因"挂在 layer_id=-1 的实体上，天然和地表种群隔离。

`ChunkData.tile_grid` 代表地表层（layer_id=0）。未来洞穴层的 chunk 数据结构待定（可能是 `ChunkData` 内嵌 `caves: dict[int, TileGrid]` 或独立 `CaveChunkData`），当前不预留字段避免过早抽象。

## 分块协调（WorldGenerator）

`WorldGenerator(seed)` 封装层 1 + 分块生成流程：

- `ensure_continent()`：主动生成并缓存 `ContinentData`（默认惰性，首次 `get_altitude` 才跑）。GameEngine 启动时主动调用
- `generate_chunk(cx, cy)`：同步生成层 1 ChunkData（群系/气候/海拔，不生成 tile）
- `generate_parallel(chunks, max_workers)`：并行生成多个分块
- `get_biome(cx, cy)` / `get_climate(cx, cy)`：快速查询

## 线程安全

| 模块 | 策略 |
|------|------|
| `noise.py` | 每个线程独立 PerlinNoise 实例（种子相同则结果相同），无共享状态 |
| `continent.py` | ContinentData 生成后不可变；ContinentGenerator 无可变状态 |
| `hydrology.py` | 侵蚀各步骤纯函数；C 模块无共享状态 |
| `streamlines.py` | build_river_network 无可变状态；RiverNetwork 构造后只读 |
| `tile_gen.py` | TileGenerator 持有独立 PerlinNoise，无共享可变状态 |
| `tile_grid.py` | TileGrid 归属单个 chunk，由该 chunk 的生成线程独占 |
| `chunk.py` | ChunkData 值对象，大地图层不可变，详细层创建后归属调用线程 |

## 加载窗口

- 玩家所在 chunk 为中心，加载 `INITIAL_CHUNK_RADIUS=2`（5×5=25 chunk）的详细 tile
- 玩家跨 chunk 边界时，移出窗口的 chunk 卸载详细层（保留大地图层），新进入的按需生成
- 纯生成的 chunk 可丢弃重算（确定性 seed 保证一致），玩家改造过的 chunk 持久化
