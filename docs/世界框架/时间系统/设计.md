# 时间系统 — 设计

## 基底时间

**2 真实小时 = 1 游戏天**。不赶人，也不慢到育种跑不动。

比传统殖民模拟游戏（20分钟/天）慢 6 倍，让日常行为（吃喝、对话、制作）有自然的呼吸感，不被高频打断。

## 四种时间模式

```
日常实时 ─→ 睡眠 ─→ 快速旅行 ─→ 长跳
正常模拟    正常模拟   加大步长     粗粒度快进
           不渲染     不渲染       不渲染
```

### 睡眠：正常模拟，不渲染

- 跳过一夜（约 8 游戏小时）。玩家点击休息后瞬间完成
- 模拟引擎保持**正常步长**——守卫继续巡逻、夜猫子 NPC 继续活动、凌晨起床的开始准备早餐
- 只是不渲染画面给玩家看。跟日常实时的区别仅在于不消耗玩家 40 分钟
- 性能完全不是问题——8 游戏小时，正常模拟也就一瞬间

### 快速旅行：加大步长

- 跳过几周到几个月。移动过程不逐帧渲染，到达时生成途中的行为记录摘要
- 模拟引擎以较大步长推进，降低事件粒度

### 长跳：粗粒度快进

- **闭关**：几十年。前提是处理好眼前事务（安排好聚落、安置好关键 NPC）
- 回来后世界已变：育种结果可见、凡人代际更替、聚落扩张或衰落
- 长跳是稀有操作，有仪式感——按下"闭关"前玩家会确认一切妥当
- 模拟以最大步长推进，按"人生事件"粒度

## 寿命梯度

| | 凡人种 | 长生种 | 修仙者 |
|---|---|---|---|
| 寿命 | 几十年 | 数百年 | 理论上无限（随境界延长） |
| 生育率 | 正常 | 低 | 极低或不育 |
| 分布 | 主要人口 | 聚居特定聚落 | 少量散布于智慧生物中 |
| 角色 | 育种代际来源 | 社会记忆锚点 | 玩家建立深度关系的跨时代同伴 |

### 修仙者境界

修仙者只修长生，不修武力。境界越高寿命越长。境界划分待细化——但早期只需 2-3 层：筑基（寿 200）、金丹（寿 500）、元婴（寿 1000+）。

玩家作为修仙者，在漫长寿命中见证凡人代际更替、社会演化兴衰。

## 快进模拟

### 原则：同一套模拟引擎，不同粒度

快进不是另一套系统——是模拟引擎加速跑完，不渲染、不逐帧展开。NPC 依然用自己的技能、朝着自己的目标前进，撞上其他 NPC 时自然产生共享事件，规模够大时自然升级为世界事件。

没有"结婚概率表"——是 NPC A 和 NPC B 各自追求目标的过程中产生交集，关系值到位、人格匹配，自然走到一起。

### 生成顺序

```
1. 外部世界事件（天灾等——不需要 NPC 参与，先确定主干）
2. NPC 个人轨迹（沿时间线推进，能力+目标+人格驱动）
3. 共享事件同步（一方产生 → 自动写入所有参与方）
4. 个人事件升级（超过显著性门槛的纳入世界事件表）
5. 受影响 NPC 以新世界事件为参照重新调整
```

### 共享事件同步

结婚、决斗、交易等涉及多方的共享事件，一方生成时自动同步到所有参与方的时间线。同一条数据，各自关系页面上有各自视角的描述。

冲突解决：若目标 NPC 的时间线上已存在冲突事件（如 B 第 6 年要远行，A 要跟 B 结婚），由人格参数决定结果——可能一方让步、双方妥协、或产生撕裂事件（"婚礼上逃婚"）。

### LLM 调用

快进期间，NPC 一生中只有少数节点真正需要 LLM 决策——重大人生抉择、关系破裂、生命威胁——在因果链分叉点、不做决策就无法继续推演该 NPC 后续轨迹时唤醒。

几十年跨度内，一个聚落只有部分 NPC 会触发 LLM 调用。具体调度的优先级和上限留待 MVP 后根据实测确定。

### 回来后

一切 LLM 对话需要的历史上下文——世界事件、人生轨迹、共享事件的参与方视角——都以结构化摘要形式存储。玩家找任何 NPC 对话时，LLM 直接检索相关摘要，开销和平时对话一致。不会因为跳了 50 年而每个 NPC 都唤醒一次 LLM。

## 与其他模块的接口

- → **生命个体**：时间驱动生理需求衰减；长跳后 NPC 在时间线上的健康变化由生理系统判定
- → **基因系统**：长跳是育种循环的时间基础——凡人几代更替后可见性状遗传结果
- → **心智系统**：快进后 NPC 的人生摘要作为 LLM 上下文；待回顾决策点触发 LLM 唤醒
- → **群体社会**：关系图谱随时间演变；世界事件作为所有 NPC 共享的记忆参照
- → **世界框架/事件系统**：世界事件是快进模拟的生成基础；快进期间模拟的小粒度事件可能升级为新的世界事件
