# 时间系统 — 技术

## 时钟实现

- 基底时间 2h = 1 游戏天，时钟以真实秒为单位推进
- 游戏内时间精度到"游戏分钟"，1 游戏分钟 = 120 真实秒
- 暂停时世界时钟停止，所有 NPC 和生理计时冻结

## 时间模式实现

### 实时（REALTIME）

- 步长 = real_dt × 12（2 真实小时 = 1 游戏天）
- WorldClock.tick(real_dt) 按真实帧间隔推进
- 发布 game_minute 事件有节流：至少间隔 1 游戏分钟

### 睡眠（SLEEP）

- 固定步长 60 游戏秒/tick
- WorldClock.fast_forward(target, mode=SLEEP) 以该步长迭代推进
- 跳过区间内不渲染，但事件正常产生
- 约 8 游戏小时 = 480 游戏分钟，480 次 tick 完成（每 tick 发布 1 个 game_minute 事件）

### 快速旅行（FAST_TRAVEL）

- 固定步长 60 游戏秒/tick（当前与 SLEEP 相同，后续可能调整为更大步长）
- 跳过区间内以降低的事件粒度推进，生成行为摘要
- 不保留逐帧细节

### 长跳（LONG_JUMP）

- 固定步长 1 游戏年/tick（31536000 秒）
- 按"人生事件"粒度推进
- 推进流程分五步，详见设计文档
- 共享事件通过关联 ID 同步到所有参与 NPC 的时间线

## 性能

- 睡眠：正常步长，毫秒级完成，无性能问题
- 快速旅行和长跳的模拟推进走同一引擎，参数不同（步长、事件粒度）
- LLM 调用在快进期间仅于因果链分叉点触发，日常行为全自动
- 具体 LLM 调度策略留待 MVP 后细化
