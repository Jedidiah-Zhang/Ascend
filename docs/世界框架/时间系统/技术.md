# 时间系统 — 技术

## 时钟实现

- 基底时间 2h = 1 游戏天，时钟以真实秒为单位推进
- 游戏内时间精度到"游戏分钟"，1 游戏分钟 = 120 真实秒
- 暂停时世界时钟停止，所有 NPC 和生理计时冻结

## 时间模式实现

### 睡眠

- 世界时钟直接推进到次日早晨，不改变模拟步长
- 跳过区间内，模拟引擎以正常步长继续运行，仅跳过渲染
- 守卫、夜行 NPC 的夜间行为照常生成行为记录
- 约 8 游戏小时 = 480 游戏分钟，正常步长下毫秒级完成

### 快速旅行

- 世界时钟推进到到达时间，模拟引擎切换为较大步长
- 跳过区间内以降低的事件粒度推进，生成行为摘要
- 不保留逐帧细节

### 长跳（快进）

- 世界时钟推进数十年，模拟引擎切换为最大步长，按"人生事件"粒度
- 推进流程分五步，详见设计文档
- 共享事件通过关联 ID 同步到所有参与 NPC 的时间线

## 性能

- 睡眠：正常步长，毫秒级完成，无性能问题
- 快速旅行和长跳的模拟推进走同一引擎，参数不同（步长、事件粒度）
- LLM 调用在快进期间仅于因果链分叉点触发，日常行为全自动
- 具体 LLM 调度策略留待 MVP 后细化
