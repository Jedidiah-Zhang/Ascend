# 标签与信息

基因差异对个体的影响，最终落在两个层面：**数值化的需求/倾向**（驱动行为决策），**标签化的状态**（管理系统复杂度）。数值是 LLM 决策的真正输入，标签是数值变动的管理信号。

## 三层结构

```
数值层（行为决策的输入，0-100 或 0-1 连续值）
  → 标签层（状态管理：触发判定、互斥、过期、长期追踪）
    → LLM 上下文组装（数值快照 + 关键标签概括）
```

- **数值层**是真实状态。LLM 能自然理解连续数值——"饱腹 17/100"比"饥饿"更精确，"社交倾向 0.62"让每次行为围绕中间值自然波动而非每次极端合群
- **标签层**是管理工具。负责：什么数值触发什么信号、哪些状态互斥、何时过期、长期趋势追踪
- **上下文组装**以数值为主，标签只是概括提示（如"社交倾向 0.72，在群体中被认知为合群倾向"）

## 标签实体

```json
{
  "id": "severe_hunger",
  "category": "physiological",
  "duration": 3600,
  "visibility": "hidden"
}
```

标签不区分长短——统一结构，`duration` 为 null 即无时限（长期标签）。

## 可见性三级

| 可见性 | 含义 | 来源示例 |
|---|---|---|
| `visible` | 视野内可直接观察 | 重伤、跛行、发抖、怀孕、在采集、在战斗 |
| `known` | 玩家已知，有确认时间戳 | 玩家亲手编辑的基因标签、亲历的事件 |
| `hidden` | 不可直接见，需调查 | 心理倾向、经历标签、技能标签 |

- **`visible`** 受遮挡判定。NPC 在屋内/墙后不可见。NPC 之间也能互相"看到"对方的 visible 标签（系统做视野判定）
- **`known`** 是玩家曾确认的快照，不会自动同步。NPC 那端标签变了，玩家这边的 known 不变。UI 上最近确认（1 天内）正常显示，较早确认的标签颜色变淡，暗示可能已过时
- **`hidden`** 标签可通过对话、档案室查询、长期观察转为 known

## 玩家 known 标签存储

玩家对每个认识的 NPC 维护一个 known 标签列表。存储复杂度 O(N)，每个 NPC 几十字节，不构成负担。

NPC **不**存储其他 NPC 的 known 标签。NPC 对其他个体的认知来自三个已有渠道：
- 关系页面（关系值 + 关系来源 + 描述）——受该 NPC 基因染色过的主观看法的压缩形式
- 行为记录——交互历史的具体事件记忆
- 视野内对方的 visible 标签

意识转移时：玩家自己的 known 列表随意识带走（扣除转移损失部分），原 NPC 的意识消散。不会出现"继承 NPC 记忆中的 known 标签"——意识是玩家自己的，不是身体的。

## 标签的管理职责

标签不做行为决策（那是数值的事），只做系统侧管理：
- **触发判定**：饱腹 < 20 → 系统标记 `severe_hunger`，通知其他模块
- **互斥**：不能同时 `freezing` 和 `overheating`
- **叠加追踪**：连续 N 天标记 `severe_hunger` → 生成长期标签 `chronic_malnutrition`，反过来影响健康数值的恢复速率
- **事件门控**：某事件要求"目标有 `skilled_gatherer` 标签"才触发
