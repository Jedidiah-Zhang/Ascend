# 目标与愿望

目标与愿望系统定义 NPC **想做什么**。它是动机层（实时需求）和执行层（具体动作）之间的持久化中间层——让 NPC 在 LLM 调用之间保持行为方向的一致。

与相关系统的关系：
- **生理/心理需求**是目标的**原料**——饥饿产生"找食物"目标
- **技能列表**是目标的**过滤器**——只保留 NPC 有能力执行的目标
- **人格**是目标的**塑形器**——同一需求，不同人格产生不同形态的目标
- **LLM** 是目标的**创作者**——宏大或定性的目标由 LLM 在反思/决策时设定

## 目标三层结构

| 层级 | 数量 | 来源 | 生命周期 | 例子 |
|---|---|---|---|---|
| **当前目标** | 1个 | 动机层自动生成 或 LLM 设定 | 完成/中断后替换 | "采集食物"、"建造木屋" |
| **待办目标** | 2-3个 | LLM 规划时设定 | 数天（游戏内） | "为冬天储备粮食"、"修复与A的关系" |
| **长期愿望** | 无上限 | LLM 反思时生成 | 数周甚至一生 | "成为族群首领"、"探索世界边界" |

当前目标和待办目标进入排序队列参与行为决策，长期愿望不参与日常排序——它们只在 LLM 反思时被审视和更新。

## 目标数据结构

```
Goal:
  id:           "goal_042"
  label:        "采集食物"               ← 自然语言，直接给 LLM 用
  category:     "survival" | "social" | "crafting" | "role" | "aspiration"
  priority:     0.85                     ← 动机层计算，含需求紧迫度
  deadline:     "immediate" | "today" | "this_week" | "this_season" | "indefinite"
  source:       
    type:       "need_driven" | "llm_set" | "role_maintenance"
    detail:     {need: "饱腹", urgency: 0.83}
  frustration:  2                        ← 连续受阻次数
  progress:     0.3                      ← 可量化进度，0-1
  prerequisites:                         ← 可选，目标成立的前置条件
    - {type: "has_skill", value: "酿造"}
    - {type: "has_item", value: "空置房屋"}
```

### 排序规则

先按 deadline 分组，组内按 priority 降序排列：

```
排序优先级：immediate > today > this_week > this_season > indefinite

同 deadline 组内：priority 高的排前面
同 priority：frustration 高的排前面（持续受阻需要 LLM 关注）
```

生存应急（deadline=immediate, priority=0.95）天然高于长期规划（deadline=indefinite, priority=0.6），无需额外调度逻辑。

## 目标生成的三个通道

### 通道一：需求驱动（系统自动）

生理/心理需求下降到阈值以下时，系统自动生成目标：

```
饱腹 < 30 → 生成目标"获取食物"，priority=1.0 - 饱腹/100，deadline=immediate
社交满足 < 20 → 生成目标"找人互动"，priority=0.6，deadline=today
```

这个通道不需要 LLM 介入。目标的具体实现方式由技能列表过滤 + 人格塑形决定。

### 通道二：角色维护（系统自动）

NPC 的持久角色身份持续产出子目标：

```
角色"酒馆老板" 自动产出：
  - "补充麦芽库存"（库存 < 阈值时触发）
  - "招一个帮手"（客流量 > 单人处理能力时触发）
  - "研发新酒品"（周期性触发，受求新性调节）
```

角色维护目标有 deadline 但通常不急迫，被生存需求覆盖时自动让步。

### 通道三：LLM 生成（反思/决策时）

LLM 在反思或重大决策时，可以创建、修改、删除目标：

- 长期愿望："我这辈子想成为族群首领"
- 创造性目标："我想研究一种新的酿酒方法"
- 关系目标："我想修复和 NPC-A 的关系"
- 放弃目标：受阻过久时，LLM 决定放弃并生成替代目标

## 目标变更的六种路径

| 路径 | 触发条件 | 结果 | 需要 LLM？ |
|---|---|---|---|
| **完成→转化** | progress = 1.0 | 目标删除。如果该目标关联了角色身份，角色创建/激活 | 否，系统自动 |
| **受阻→反思** | frustration > 阈值（默认 3） | LLM 决定：换方法 / 降低目标 / 彻底放弃 | **是** |
| **条件丧失** | prerequisite 中的条件永久不满足 | LLM 重新评估可行性 | **是** |
| **优先级覆盖** | 生存危机/紧急事件 | 当前目标暂停，插入紧急目标。原目标进入待办列表 | 否，系统自动 |
| **人格漂移** | 高求新性 NPC 长期做重复工作 | LLM 决定：寻找新目标 / 继续坚持 | **是** |
| **机会冲击** | 发现重大新资源/新可能 | LLM 比较新旧目标，决定是否转向 | **是** |

### 受阻检测

```
frustration 每次技能执行失败 +1
frustration 每次技能执行成功 归零
frustration 连续 ≥ 3 → 唤醒 LLM："这个目标失败了 3 次，你打算怎么办？"
```

LLM 的回应写入目标记录，可能是"换个地方钓鱼"（修改方法），"减少目标量级"（降低目标），或"算了不干了"（删除目标 + 生成替代）。

### 条件丧失检测

系统定期（每天）检查所有待办目标和长期愿望的 prerequisites。如果任一条件永久不满足（技能被遗忘且无法重学、所需物品被摧毁且不可再生），标记该目标为"待复查"。下次 LLM 周期性反思时一并处理。

## 目标→角色转化

当一个目标完成时，如果它本质上是身份性的（"开酒馆"、"成为首领"），目标删除后自动创建对应的**角色**：

```
目标"开设酒馆" progress=1.0
       │
       ▼  系统检测：目标 label 匹配身份类关键词
       │
       ├──→ 目标"开酒馆"从愿望列表移除
       └──→ 角色"酒馆老板"添加到 NPC 身份
                │
                ▼  角色自动生成子目标：
                     - "补充麦芽库存"
                     - "招一个帮手"
                     - "让更多人知道酒馆开业"  ← LLM 推导，非手工预设
```

角色是目标的"完成态"——目标是一次性的，角色是持续性的。角色本身也会被重新审视（LLM 反思时可以决定放弃角色），详见角色生命周期。

## 长期愿望的保鲜机制

长期愿望不会被遗忘，但会被**复查**。复查分为两个级别：

| 复查级别 | 触发条件 | 处理方式 |
|---|---|---|
| **系统级** | prerequisite 中的条件发生变化 | 标记为"待复查"，不立即唤醒 LLM |
| **LLM 级** | 周期性反思时检查所有"待复查"项 | 合并到一次 LLM 调用中处理 |

此外，长期愿望有一个隐性的"自然淘汰"机制：当愿望的 prerequisites 在多次复查后都无法满足，LLM 自行决定是否放弃。系统不做自动删除——只有 LLM 能决定放弃自己曾经设立的愿望。

## 目标冲突裁决

当两个高优先级目标互相排斥时（"照顾生病的孩子" vs "接待大客户"），由 LLM 根据性格裁决：

```
LLM 输入：
  当前有两个冲突目标：
  - 目标A："照顾生病的孩子"（孩子高烧，不处理可能恶化）
  - 目标B："接待大客户"（决定酒馆本月收入）
  
  你的性格维度：抚育性 0.75，勤勉性 0.6，支配性 0.5
  
  请选择一个执行，并给出另一个目标的后续处理方式（挂起/放弃/委托他人）。
```

裁决结果写入：执行层执行选中的目标，另一个目标按 LLM 的决定处理（挂起写入待办列表、放弃则删除、委托则转为社交目标"找 X 帮忙照顾孩子"）。

## 与 LLM 的交互边界

| 场景 | 系统处理 | LLM 处理 |
|---|---|---|
| 饿了找食物 | 自动生成目标 + 技能过滤 + 人格选择 | 不参与 |
| 饿了但我只会钓鱼和偷窃，选哪个 | 人格加权决策 | 不参与 |
| 连续偷窃失败 3 次，还继续吗 | — | 裁决 |
| 邻居告诉我山里有新食物源，去不去 | — | 判断 |
| 我开了三年酒馆，想换个活法 | — | 反思时生成新愿望 |
| 我想成为族群首领 | — | 反思时生成长期愿望 |
