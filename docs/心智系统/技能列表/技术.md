# 技能列表 — 技术实现

## 数据存储

技能模板为 JSON 数据文件，按类别分目录：

```
data/skills/
  gathering/    钓鱼.json, 采集.json, 狩猎.json, ...
  crafting/     酿造.json, 木工.json, 烹饪.json, ...
  social/       谈判.json, 说服.json, 激励.json, ...
  combat/       格斗.json, 弓箭.json, ...
  management/   调配物资.json, 裁决纠纷.json, ...
```

每个 NPC 实例存储已掌握的技能引用 + 熟练度：

```json
{
  "learned_skills": [
    {"skill_id": "fishing", "proficiency": 0.73},
    {"skill_id": "gathering", "proficiency": 0.45}
  ],
  "innate_skills": ["grasping", "bipedal_movement"],
  "max_learned": 5
}
```

先天技能不存熟练度（始终可用，不受熟练度影响），角色技能独立存储在角色数据中。

## 技能展开：执行层的翻译机制

执行层不硬编码每个技能的展开方式。技能模板中的 `preconditions` 是声明式的，执行层有一个通用的**前置条件求解器**：

```
function expand_skill(skill, world_state):
    actions = []
    
    for precondition in skill.preconditions:
        if not satisfied(precondition, world_state):
            sub_actions = resolve_precondition(precondition, world_state)
            actions.extend(sub_actions)
    
    actions.append(skill.action_sequence)
    return actions
```

`resolve_precondition` 的类型：

| 前置条件类型 | 解决动作 |
|---|---|
| `tool: [钓竿]` | 从背包装备钓竿；如果没有→去仓库取；如果仓库没有→制作 |
| `location: [水域]` | 寻路到最近的匹配水域 tile |
| `state: {体力: ">10"}` | 等待/休息直到满足；如果持续不满足→向上汇报无法执行 |

只有当所有前置条件都能满足时，技能才会被执行。如果前置条件链中某一环无法满足（如没有水域可到达），技能候选被排除，回到技能过滤环节选择下一个方案。

## 熟练度计算

熟练度影响技能执行的两个参数：

```
actual_duration = base_duration × (1.0 - proficiency × 0.5)
                 × random(1.0 - variance, 1.0 + variance)

actual_output_quality = base_output × (1.0 + proficiency × 0.5)
```

即熟练度从 0 到 1，duration 从 100% 降到 50%，output 从 100% 升到 150%。

熟练度增长：

```
proficiency_gain_per_use = base_gain × (1.0 - proficiency) × cognitive_factor

base_gain = 0.02
cognitive_factor = 由基因中的"学习效率"参数决定
```

越熟练增长越慢，接近上限时需要大量实践。

## 遗忘机制

当已习得技能数达到上限且需要学习新技能时：

```
遗忘速率 = base_decay × (1.0 - proficiency)
          × 活跃抑制因子（最近使用过的技能遗忘更慢）

base_decay = 0.01 / 天
```

- 久未使用的技能自然衰减，到 0 后自动从 `learned_skills` 移除
- NPC 会倾向于遗忘使用频率最低的技能（自然机制，不需要显式选择）
- LLM 设定学习目标时，如果技能已满，LLM 需要在 prompt 中指出要放弃哪个技能
