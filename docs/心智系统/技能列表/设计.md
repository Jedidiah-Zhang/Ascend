# 技能列表

技能列表定义 NPC 的能力边界——**能做什么**。它是动机层产出目标后、执行层采取行动前的过滤器：只有 NPC 具备对应技能的目标才能进入候选。

与目标系统的关系：
- **目标/愿望列表**决定 NPC **想做什么**（动机方向）
- **技能列表**决定 NPC **能做什么**（能力边界）
- 需求产生目标，技能过滤方案，人格塑形选择

## 技能即 Agent Skill

每个技能是一个可被执行层直接展开的原子单元。数据结构参照 Agent 框架的 skill 定义：

```
Skill:
  name:         "钓鱼"
  description:  "在水域使用钓竿获取鱼类"
  category:     "采集" | "制造" | "社交" | "战斗" | "管理"
  preconditions:
    tool:       [钓竿]           ← 可选，所需工具
    location:   [水域]           ← 可选，所需位置类型
    state:      {体力: ">10"}    ← 可选，自身状态要求
  duration:     {base: 120, variance: 60}  ← 秒，受熟练度调节
  output:       [{item: 鱼, amount: 1-3, probability: 0.9},
                 {item: 垃圾, amount: 1, probability: 0.1}]
  costs:
    fatigue:    15
    tool_wear:  {钓竿: 1}
```

### 设计原则

- **preconditions 是执行层的指令**——执行层拿到一个技能后，先检查 preconditions，不满足则自动生成前置动作序列（拿钓竿 → 走到水边 → 开始钓鱼）
- **output 是概率化的**——不是"每次钓到 2 条鱼"，而是概率分布，制造真实世界的不确定性
- **duration 表达为范围**——base ± variance，再由熟练度缩放，避免机械重复感

## 技能来源三层

| 来源 | 说明 | 获取方式 | 例子 |
|---|---|---|---|
| **先天** | 基因和身体结构决定的 | 出生时自动获得 | 有翅膀→能飞、有鳃→水下呼吸、有手→能抓握 |
| **习得** | 反复实践积累的 | 执行次数达标后自动获得，受认知能力限制 | 酿造、木工、识字、烹饪、谈判 |
| **角色赋予** | 社会角色带来的权限操作 | 被任命/选举时获得，角色失去时移除 | 仓库管理员→调配物资、法官→裁决纠纷 |

- 先天技能是硬边界——没有对应身体部位，相关技能根本不可用
- 习得技能有熟练度（0-1），影响 duration 和 output 质量
- 角色技能不是真正的"技能"，而是**权限**——解锁了对特定物品/设施/决策的操作

## 认知能力限制习得数量

NPC 的基因决定了认知能力上限——不是所有 NPC 都能学会所有技能：

```
max_learned_skills = base_capacity + gene_bonus

base_capacity = 3（所有智慧生物的基础值）
gene_bonus = 由基因中的"认知"相关基因决定（0-10）
```

当已习得技能数达到上限时，学习新技能需要**遗忘**一个旧技能（熟练度缓慢衰减，降到 0 后移除）。这制造了 NPC 之间的能力分化——没有人能成为全才。

## 技能在决策流中的位置

```
动机层输出候选目标（"获取食物"）
       │
       ▼
技能列表过滤：NPC 有哪些获取食物的技能？
       │  ["采集", "钓鱼", "偷窃"]  ← 只有这些可选
       │  （"狩猎"不可选——NPC 没有狩猎技能）
       ▼
人格塑形选择：在可用方案中，人格倾向选择哪个？
       │  高耐心 + 低斗争性 → 优先"钓鱼"
       │  低耐心 + 高斗争性 → 优先"偷窃"
       ▼
执行层展开：将"钓鱼"展开为动作序列
       │  检查 preconditions → 拿钓竿 → 去水边 → 抛竿 → 等待 → 收竿
       ▼
执行并记录结果
```

## 技能的 LLM 交互

技能列表本身是系统驱动的，但 LLM 在两种场景下与技能列表交互：

1. **规划时**：LLM 设定长期目标（"我想开酒馆"）→ 系统检查 NPC 是否具备所需技能（酿造）→ 如果不具备，LLM 生成习得计划（"先找个会酿造的人教我"）
2. **受阻时**：技能执行连续失败 → LLM 决定是否换一个方法或放弃目标

日常的技能选择（"饿了，我会钓鱼和采集，选哪个"）不需要 LLM 参与——系统用技能过滤 + 人格加权自动决策。
