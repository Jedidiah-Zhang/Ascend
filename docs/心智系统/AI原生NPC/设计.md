# AI 原生 NPC

**本游戏的核心差异化定位：NPC 默认由 LLM 驱动。** 行为、对话、决策、记忆全部 AI 原生，而非预设行为树和对话表。基因系统、记录系统、关系图谱等所有子系统存在的意义，首先是为 AI NPC 提供结构化、可检索的上下文。

市面上现有游戏的 AI NPC mod（RimWorld、Minecraft、鬼谷八荒等）的共同缺陷是：LLM 作为外挂接入，只能看到"此刻的快照"——没有持续性人格、没有长期记忆、行为与对话脱节。本游戏从架构层面解决这个问题：**AI 是一等公民，所有游戏系统向其输送上下文。**

## AI 架构：五层 + LLM

```
                    ┌─────────────────────────────┐
                    │        LLM 决策层            │
                    │  长期规划 / 重大抉择 / 对话   │
                    │  角色创建与放弃 / 愿望生成    │
                    └──────────┬──────────────────┘
                               │
              ┌────────────────┼────────────────┐
              ▼                ▼                ▼
      ┌───────────┐   ┌───────────┐   ┌───────────┐
      │  角色层    │   │ 目标/愿望  │   │ 技能列表   │
      │ 我是谁     │   │ 想做什么   │   │ 能做什么   │
      └─────┬─────┘   └─────┬─────┘   └─────┬─────┘
            │               │               │
            └───────────────┼───────────────┘
                            │ 目标 + 候选方案
                            ▼
              ┌─────────────────────────┐
              │   动机层 (需求优先级)     │
              │   生理需求 + 心理满足度   │
              │   人格塑形方案选择        │
              └──────────┬──────────────┘
                         │ 最终选定动作
                         ▼
              ┌─────────────────────────┐
              │   执行层 (状态机)         │
              │   动作序列展开与执行      │
              │   ┌─────────────────┐   │
              │   │ 记忆模块 (RAG)   │   │
              │   └─────────────────┘   │
              └─────────────────────────┘
```

- **LLM 决策层**：根据结构化上下文生成 NPC 的长期目标、计划和对话。不是每帧调用——仅在显著性评分超过阈值时触发
- **角色层**：持久身份，持续产出维护性子目标。目标完成时自动转化（"开酒馆"→角色"酒馆老板"）。角色可嵌套并存，也可被 LLM 放弃
- **目标与愿望层**：1 个当前目标 + 2-3 待办 + N 个长期愿望。按 priority + deadline 排序。简单需求目标系统自动生成，宏大目标 LLM 设定
- **技能列表**：NPC 的能力边界（先天/习得/角色赋予）。在需求产生目标后、执行前过滤候选方案。只有 NPC 具备对应技能的目标才能进入执行
- **动机层**：基于生理需求和心理满足度计算目标 priority，按人格塑形方案选择（同一"获取食物"目标，不同人格选择不同实现方式）
- **执行层**：状态机将选定的技能展开为动作序列。技能模板的 preconditions 声明式定义前置动作，执行层自动求解
- **记忆模块（RAG 架构）**：行为记录和交互历史转为 embedding，LLM 调用时检索相关记忆参与决策

## 结构化上下文：为 AI 输送"我是谁"

NPC 调用 LLM 时，上下文窗口中的内容不是"把各种文本拼在一起"，而是从各子系统中组装的**结构化数据**：

| 子系统 | 为 AI 上下文提供 |
|---|---|
| 基因 | 不可改变的性格底色：社交偏好（外向/孤僻）、冲动程度、能力倾向。不是"描述文本"，是参数约束 |
| 关系图谱 | 该 NPC 对对话对象的看法（关系值 + 关系来源 + 简短描述），决定态度和语气 |
| 行为记录 | 近期记忆素材——"昨天下午我在河边的树林采集药草，收获 12 份"。结构化记录 → 自然语言化后送入 LLM |
| 动机层 | 当前需求状态（饥饿、安全焦虑、社交渴望），影响对话时的关注点和行为倾向 |
| 目标与愿望 | 正在追求的目标列表和长期愿望，为 LLM 提供"我正在做什么、我想达成什么"的持续上下文 |
| 技能列表 | NPC 的能力清单，LLM 规划时作为能力约束——"我想飞过峡谷"→检查技能→"我不会飞，换个方案" |

每个 NPC 的 AI 上下文不是一次性构建的——在不同场景下动态组合不同子集：对话时侧重关系+动机，决策时侧重基因+记忆+动机+目标+技能。

## 对话即行为

NPC 对话不是对话树。每次对话是 LLM 的一次推理，输入是结构化上下文，输出是自然语言。但对话**本身**就是会产生记录的行为——对话结束后：

- 对话摘要写入双方的行为记录
- 关系值根据对话内容和结果微调
- 如果对话中包含了关键信息（如 NPC-A 告诉玩家 NPC-B 的看法），玩家的信息不对称被更新

这意味着 NPC 的"说的话"和"做的事"是同一个人格的两面——不会出现"嘴上说信任你但行为背叛你"的割裂感，除非它的基因就是这样（高欺骗倾向）。

## 已有系统如何服务 AI NPC

此前的所有设计——基因、记录驱动模拟、关系图谱、信息不对称——在此框架下统一了目标：

- **基因系统**：为每个 NPC 提供不可改变的"性格底色"，是 LLM 长期一致性的锚点。同一个事件，不同基因的 NPC 产生不同的记忆记录和对话回应
- **记录驱动模拟**：视野内 NPC 的行为产生真实记录，视野外 NPC 产生概率记录。所有记录都是结构化、可检索的——这就是 AI 的"长期记忆"
- **关系图谱**：为 LLM 提供社会上下文。NPC 跟你说话时的态度，取决于它对你的关系页面上写了什么
- **信息不对称**：你不能直接查询 NPC-A 对 NPC-B 的看法——你必须通过对话去问。AI 驱动的 NPC 使这种"社交获取信息"的玩法变得不可预测、不可穷尽

## LLM 唤醒：显著性评分

LLM 不应该手工枚举"何时该唤醒"——那是无底洞。用统一的显著性评分机制，让系统自动判断一个事件是否值得唤醒 LLM。

### 三层评分

```
第一层：事件分类（5档，全局一致）

  濒死级 1.0  │  自身或至亲生命受威胁、族群存亡危机
  重大级 0.8  │  重要资源变化、社会地位变动、关系突变
  关注级 0.5  │  邻里八卦、小型机会、日常异常
  日常级 0.2  │  重复事件、小幅波动
  背景噪音 0.0│  与己无关的远方事件、无影响的自然变化

第二层：NPC 人格过滤

  人格决定某类事件"对他而言"是否重要。不是乘系数——是直接映射：

  事件类型          │ 在意的 NPC                 │ 无视的 NPC
  ─────────────────────────────────────────────────
  社交冲突           │ 高社交性、高斗争性          │ 低社交性、孤僻
  新事物/新机会      │ 高求新性                   │ 低求新性
  秩序/规则变动      │ 高秩序性                   │ 低秩序性（随性）
  弱者遭难           │ 高抚育性                   │ 低抚育性
  远期趋势           │ 高时间视野                 │ 低时间视野

  映射结果：该 NPC 对此事件类型 = "在意"/"无视"/"反向在意"

第三层：状态抑制

  唯一规则：

  "生存危机中"（饱腹<20 或 生命受威胁）
    → 所有非生存类事件降为背景噪音
    → 只有 濒死级 和 生存相关的重大级 事件不被抑制
```

### 最终判定

```
第一层 → 第二层映射后 → 第三层抑制后 → 最终显著性

最终显著性 ≥ NPC 关注阈值 → 唤醒 LLM
最终显著性 < NPC 关注阈值 → 系统自动处理

关注阈值由人格基线决定：
  高求新性 → 阈值低（频繁反思，容易被任何事情唤醒）
  低求新性 → 阈值高（安于现状，只有大事才动脑子）
  默认基线 = 0.5
```

### 玩家告知绕过

当玩家**主动告知** NPC 某件事时，绕过显著性评分，强制唤醒 LLM。这让玩家成为信息传播者，也是解决"NPC 为什么没反应"的最终兜底手段。

### 三类触发源的配合

| 触发源 | 频率 | 说明 |
|---|---|---|
| **外部事件中断** | 事件发生时 | 经过显著性评分过滤，玩家搭话强制唤醒 |
| **目标状态变更** | 目标受阻/条件丧失时 | frustration 达阈值唤醒；条件丧失标记待复查，合并到反思时处理 |
| **周期性自我反思** | 由人格决定间隔 | 高求新性 NPC 频繁反思，低求新性安于现状。反思时一并处理所有待复查项 |

## 性能策略

LLM 调用有延迟和成本。策略：
- **仅智慧生物调用 LLM**。非智慧生物（动植物）使用传统状态机
- **视野内智慧 NPC 才触发 LLM 决策**。视野外 NPC 通过概率模型 + 基因参数生成行为记录（不调 LLM）
- **对话触发、重大决策触发、周期性反思触发**——不是每帧、不是每个动作
- MVP 阶段可先用本地小模型（如 Qwen 等），模型切换透明
