# AI 原生 NPC

**本游戏的核心差异化定位：NPC 默认由 LLM 驱动。** 行为、对话、决策、记忆全部 AI 原生，而非预设行为树和对话表。基因系统、记录系统、关系图谱等所有子系统存在的意义，首先是为 AI NPC 提供结构化、可检索的上下文。

市面上现有游戏的 AI NPC mod（RimWorld、Minecraft、鬼谷八荒等）的共同缺陷是：LLM 作为外挂接入，只能看到"此刻的快照"——没有持续性人格、没有长期记忆、行为与对话脱节。本游戏从架构层面解决这个问题：**AI 是一等公民，所有游戏系统向其输送上下文。**

## AI 架构：四层 + LLM

```
┌─────────────────────────────────────────┐
│                  决策层 (LLM)            │  长期目标、计划、对话生成
│   ┌─────────────────────────────────┐   │
│   │         动机层 (需求优先级)       │   │  欲望、生理需求 → 动作排序
│   │  ┌───────────────────────────┐  │   │
│   │  │    执行层 (状态机)         │  │   │  具体动作序列
│   │  │  ┌─────────────────────┐  │  │   │
│   │  │  │   记忆模块 (RAG)     │  │  │   │  短期/长期记忆 embedding 检索
│   │  │  └─────────────────────┘  │  │   │
│   │  └───────────────────────────┘  │   │
│   └─────────────────────────────────┘   │
└─────────────────────────────────────────┘
```

- **决策层（LLM）**：根据结构化上下文生成 NPC 的长期目标、计划和对话。不是每帧调用——仅在关键节点触发（被玩家搭话、做出重要决策、遭遇意外事件、周期性自我反思）
- **动机层**：基于基因赋予的欲望倾向和当前生理需求，将目标按优先级排序。动机层是决策层的"前置过滤器"
- **执行层**：状态机将决策转化为具体动作序列（移动到 X、采集 Y、与 Z 交谈）。不需要 LLM 参与每个步骤
- **记忆模块（RAG 架构）**：行为记录和交互历史转为 embedding，LLM 调用时检索相关记忆参与决策。短期记忆 = 最近 N 条记录，长期记忆 = embedding 相似度检索

## 结构化上下文：为 AI 输送"我是谁"

NPC 调用 LLM 时，上下文窗口中的内容不是"把各种文本拼在一起"，而是从各子系统中组装的**结构化数据**：

| 子系统 | 为 AI 上下文提供 |
|---|---|
| 基因 | 不可改变的性格底色：社交偏好（外向/孤僻）、冲动程度、能力倾向。不是"描述文本"，是参数约束 |
| 关系图谱 | 该 NPC 对对话对象的看法（关系值 + 关系来源 + 简短描述），决定态度和语气 |
| 行为记录 | 近期记忆素材——"昨天下午我在河边的树林采集药草，收获 12 份"。结构化记录 → 自然语言化后送入 LLM |
| 动机层 | 当前需求状态（饥饿、安全焦虑、社交渴望），影响对话时的关注点和行为倾向 |

每个 NPC 的 AI 上下文不是一次性构建的——在不同场景下动态组合不同子集：对话时侧重关系+动机，决策时侧重基因+记忆+动机。

## 对话即行为

NPC 对话不是对话树。每次对话是 LLM 的一次推理，输入是结构化上下文，输出是自然语言。但对话**本身**就是会产生记录的行为——对话结束后：

- 对话摘要写入双方的行为记录
- 关系值根据对话内容和结果微调
- 如果对话中包含了关键信息（如 NPC-A 告诉玩家 NPC-B 的看法），玩家的信息不对称被更新

这意味着 NPC 的"说的话"和"做的事"是同一个人格的两面——不会出现"嘴上说信任你但行为背叛你"的割裂感，除非它的基因就是这样（高欺骗倾向）。

## 已有系统如何服务 AI NPC

此前的所有设计——基因、记录驱动模拟、关系图谱、信息不对称——在此框架下统一了目标：

- **基因系统**：为每个 NPC 提供不可改变的"性格底色"，是 LLM 长期一致性的锚点。同一个事件，不同基因的 NPC 产生不同的记忆记录和对话回应
- **记录驱动模拟**：视野内 NPC 的行为产生真实记录，视野外 NPC 产生概率记录。所有记录都是结构化、可检索的——这就是 AI 的"长期记忆"
- **关系图谱**：为 LLM 提供社会上下文。NPC 跟你说话时的态度，取决于它对你的关系页面上写了什么
- **信息不对称**：你不能直接查询 NPC-A 对 NPC-B 的看法——你必须通过对话去问。AI 驱动的 NPC 使这种"社交获取信息"的玩法变得不可预测、不可穷尽

## 性能策略

LLM 调用有延迟和成本。策略：
- **仅智慧生物调用 LLM**。非智慧生物（动植物）使用传统状态机
- **视野内智慧 NPC 才触发 LLM 决策**。视野外 NPC 通过概率模型 + 基因参数生成行为记录（不调 LLM）
- **对话触发、重大决策触发、周期性反思触发**——不是每帧、不是每个动作
- MVP 阶段可先用本地小模型（如 Qwen 等），模型切换透明
