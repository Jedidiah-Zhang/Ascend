# Ascend — 研究方案与理论框架 (v0.1)

## 1. 研究动机

当前因果表征学习 / 世界模型领域存在一个根本困境：**缺乏可验证的 ground truth**。在真实数据上训练的模型声称"学到了因果结构"，却无法用反事实干预来证明——你无法干预现实世界。

另一方面，程序化生成、物理因果仿真、结构化事件日志等技术已成熟，却鲜有被有意识地面向"世界模型训练"进行系统设计。

Ascend 试图填补该交叉领域：以可复现、可干预的因果世界为平台，提供带已知 ground truth 的标准测试环境，研究世界模型在结构化因果空间中的学习行为。

---

## 2. Ascend 的独特研究资产

在因果 ML 研究语境下，Ascend 提供三项不可替代的能力：

| 资产                                | 研究价值                                                           |
| ----------------------------------- | ------------------------------------------------------------------ |
| **已知因果 DAG（WorldTree）** | 所有事件间的因果关系在代码中显式声明，为正例学习提供强先验         |
| **完整时空序列日志**          | 每个因果事件携带父节点数值快照，可构成监督训练样本                 |
| **可干预、可复现**            | seed + 初始状态配置 + headless 批量运行，实现任意 do-operator 干预 |

核心研究假设：**从已知 DAG + 时空序列中，世界模型可以学到 ε-精度的因果映射，且在有 DAG 先验时样本复杂度由最大因果入度决定，而非变量总数。**

## 3. 理论框架

### 3.1 结构因果模型（SCM）

SCM 用数学方程描述"什么导致什么"，而非仅描述相关性。每个变量由一个结构方程决定，其输入是该变量的全部直接原因。

```
因果 DAG (WorldTree) + 结构方程 = SCM

    气温                 气温 = f₁(纬度, 海拔)
     ↓
    降水                  降水 = f₂(气温, 湿度)
     ↓
    河流水位              水位 = f₃(降水, 地形)
```

关键性质：方程是**不对称的**——调气温影响降水，反向不成立。这区别于相关性（对称）。

在 Ascend 中，WorldTree 提供了已知的 DAG 结构。训练世界模型的问题退化为：给定因果图，拟合结构方程。因为有 ground truth DAG，无需盲猜因果方向，问题难度大幅降低。

### 3.2 Granger 因果

Granger 因果提供时序预测层面的因果检验手段：

```
预测₁: Y(t) ≈ g₁(Y(t-1), Y(t-2), ...)
预测₂: Y(t) ≈ g₂(Y(t-1), Y(t-2), ..., X(t-1), X(t-2), ...)

若 预测₂ 显著优于 预测₁ → X Granger-导致 Y
```

**注意**：Granger 因果 ≠ 真正因果（如公鸡叫 → 太阳升起）。在封闭系统中，当所有相关变量均被观测时，Granger 因果逼近真因果。Ascend 的封闭世界满足该前提。

### 3.3 两种框架在 Ascend 中的分工

| 框架    | 角色                                           | Ascend 优势            |
| ------- | ---------------------------------------------- | ---------------------- |
| SCM     | 理论先验："我知道 DAG 结构，只需拟合方程"      | WorldTree 提供完整 DAG |
| Granger | 实验验证："从数据中检验模型是否学到了因果方向" | 时序日志提供检验数据集 |

## 4. 核心理论方向：已知 DAG 条件下的样本复杂度

### 4.1 问题定义

给定已知因果 DAG，学习器需要多少条因果链样本，才能使世界模型对所有结构方程达到 ε-精度（置信度 1-δ）？

### 4.2 核心洞察：DAG 带来的降维

已知 DAG 时，联合分布可分解为独立子问题：

```
P(降水 | 气温, 湿度) × P(水位 | 降水, 地形) × P(NPC行为 | 水位, 目标, 记忆) × ...
```

每个节点只依赖其父节点（WorldTree 已标注）。每个条件分布**独立学习**，整体复杂度由入度最大的节点决定：

```
无 DAG: m ∝ O(exp(n))         n = 变量总数
有 DAG: m ∝ O(exp(k))         k = max indegree
```

Ascend 的因果链是稀疏的（物理因果路径通常不长），k 天然可控。

### 4.3 待推导的三个子问题

**子问题 1：有 DAG vs 无 DAG 的样本差距**

- 无 DAG 学习联合分布，样本复杂度随变量数指数增长
- 有 DAG 分解为局部条件分布，复杂度仅取决于最大入度
- Ascend 可同时跑两种训练模式，实验验证理论差距

**子问题 2：因果路径长度与误差传播**

- 因果链 A → B → C → D，每步有 ε 拟合误差
- 经过 k 步后累积误差的增长速率
- 直接关联"感知窗口需要多深的历史"

**子问题 3：因果覆盖度判据**

- 需要多少条不同的因果路径才能保证泛化？
- 通过 do-operator 主动采样稀有因果路径，而非等自然演化

### 4.4 预期理论形式

基于 SCM + PAC-Bayes / Rademacher 复杂度框架，预期可导出一个 sample complexity bound：

```
m ≥ O( dᵏ / ε² · log(1/δ) )
```

其中 d 为变量值域，k 为最大因果入度，ε 为精度，δ 为置信度。关键洞察：有 DAG 时 k ≪ n（n = 总变量数），而无 DAG 时 k ≈ n。

## 5. 对 WorldTree / 事件系统的设计要求

理论成立依赖以下五项约束，均可在设计阶段保证：

### 5.1 DAG 完备性

每个事件节点的 cause 集合必须是该节点的**充分因果集**——不能遗漏决定该事件的变量。

```
正确: event.snow_start, cause: {temperature, humidity, wind}
错误: event.snow_start, cause: "天气变冷"（不可量化，且可能遗漏关键变量）
```

Ascend 的优势：代码逻辑是确定性的，校验 cause 是否充分相对容易——任一影响节点输出的输入变量若未出现在 cause 中，即为遗漏。

### 5.2 DAG 无环

因果边只能从时间早的事件指向时间晚的事件。每个事件携带时间戳，禁止同时刻互指。

Ascend 自然满足：物理时间单调递增，不存在"回到过去"的因果边。语义层面需警惕——如"尸体腐烂影响生态 → 影响未来天气"应建模为指向未来时间点的事件，而非回环。

### 5.3 父节点值可量化

每个事件必须携带其全部父节点在触发时刻的数值快照，构成 (X₁, X₂, ..., Xₖ → Y) 的训练样本。

要求与 5.1 协同保证：cause 中的每个键不仅是标签，更是带数值的 kv 对。

### 5.4 最大入度可控

若某节点有 20 个父节点，样本需求将指数爆炸。约束 max indegree ≤ 5~6。

Ascend 的物理系统天然低入度：天气依赖少数气象参数，生态依赖少量环境变量。需在设计事件系统时显式约束。

### 5.5 支持干预采样

自然演化可能极难覆盖某些因果路径组合。需要 do-operator 主动注入。

实现方案（无需修改生产代码）：

- **seed + 初始状态配置**：设定外生变量初始值，让因果链自然演化
- **checkpoint / restore**：存档读档，精确复现任何场景
- **headless 批量运行**：前后端分离使得无头模式自然支持

这等价于标准 do-operator 语义：`do(X=x)`，然后让结构方程自己跑。

## 6. 研究路线图

### 阶段 1：事件系统合规

确保 WorldTree 满足 §5 中的五项要求。在现有架构上进行约束性审查和补全。

### 阶段 2：基线实验

- 在已知 DAG 条件下训练世界模型，度量时序预测精度
- 用 Granger 检验验证模型学到的是因果方向而非伪相关
- 在无 DAG 条件下训练同等容量模型，量化 DAG 先验带来的提升

### 阶段 3：理论推导

- 推导已知 DAG 条件下世界模型的样本复杂度 bound
- 在 Ascend 上用不同因果路径数量做消融实验，验证 bound 的紧度
- 分析因果路径长度对误差传播的影响

### 阶段 4：反事实验证

- 用 do-operator 做干预实验：改变某个变量，验证模型预测的行为变化与引擎实际运行一致
- 量化模型在未见因果路径上的泛化能力
