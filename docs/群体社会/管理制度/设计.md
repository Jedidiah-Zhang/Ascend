# 管理制度

## 管理能力渐进路线

**管理能力是渐进获得的，而非系统赋予的身份。**

### 早期：纯个体视角

游戏开始时，玩家没有管理界面、没有种群统计面板、没有地图标记。想知道同伴在做什么？走过去看。想让人去哪？开口说。玩家与世界的关系是 1对1 的直接交互。

### 中期：制造管理工具

通过建造设施逐步获得管理能力：

- **实验室**：分析样本、提取基因、进行改造
- **议事厅/集会点**：召集NPC、口头传递规则、协商事务
- **通讯器/瞭望台**：在基地看到远处NPC的大致状态
- **记录板/档案室**：种群知识可以记录和查询

每一项管理能力的获得是物理的、有代价的、需要维护的。你"制造出"了管理工具，而不是"解锁了"系统菜单。

### 后期：制度化

- 档案系统让种群知识可以跨代传承
- 规则系统固化——NPC按规则行事，即使玩家不直接干预
- 玩家减少具体操作，进入观察和规划的角色

### 玩家角色定位

玩家偏向于"规则制定者"，而非"控制者"。

**管理能力由设施提供，不由角色绑定。** 喜欢纯个体视角的玩家，始终可以亲自外出探索，种群的决策可以交给其他 NPC 来填补。个体视角和管理视角在同一条时间线上共存，不是二选一。

## NPC 管理能力

**其他种群中也可以涌现出具备管理能力的NPC。** 管理者NPC不是预设身份（如"领袖NPC"），而是个体在以下条件同时满足时涌现：

1. **有能力**：基因组合碰巧产生了适合管理的特征——更高的认知、规划倾向、社交需求、较低的冲动
2. **有动机**：对种群当前状态不满（食物不足、秩序混乱），并相信自己能改善
3. **有机会**：种群存在管理真空——玩家没有管，别人也没管
4. **有工具**：有相关设施可供调用（议事厅、记录系统等）

管理行为的四种形态等级：

| 等级       | 条件              | 表现                                                 |
| ---------- | ----------------- | ---------------------------------------------------- |
| 口头号召   | 无特殊设施        | 对周围几个NPC喊话，关系高的可能跟随。不稳定。        |
| 非正式协调 | 有常聚集地点      | 定期组织采集队、轮班瞭望。依赖个人威望。             |
| 制度化管理 | 议事厅 + 记录系统 | 制定的规则持续生效，奖惩有记录。管理者死后规则仍在。 |
| 制度化传承 | 档案系统          | 新任管理者可读取前任决策记录，管理能力跨代积累。     |

玩家与管理NPC之间的张力是游戏的核心叙事载体：你可以信任它、与它竞争（改造自己以获得更强的管理能力）、改造它（使它更服从或更自主）、或观察它的决策。你在影响生态，而非下命令。
