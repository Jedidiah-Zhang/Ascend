# 世界模拟

**世界模拟采用记录驱动而非全量实时模拟。** 视野内的 NPC 全模拟，视野外的 NPC 不模拟具体行为，而是按概率分布生成行为记录，并依据记录更新关系图谱。

## 记录驱动的模拟

| 范围       | 模拟方式                                                                      | 开销                            |
| ---------- | ----------------------------------------------------------------------------- | ------------------------------- |
| 玩家视野内 | 全量实时模拟，产生真实行为记录                                                | 随视野内 NPC 数量增长           |
| 玩家视野外 | 不模拟具体行为。基于该 NPC 的基因、当前关系、所在区域，按概率模型生成行为记录 | 随记录条数增长，与 NPC 总数解耦 |

**视野内 = 全模拟 + 真实记录。视野外 = 概率生成记录 + 更新关系图谱。**

## 行为记录与部落纪事

每个 NPC（包括玩家）的所有行为以**记录条目**的形式存储。记录格式包含地点坐标：

```
[时间] [行为者] [行为类型] [目标/对象] [位置坐标] [结果摘要]
```

示例："第 47 天下午，阿卡在 (128, -304) 采集药草，收获 12 份。"

世界使用类似 Minecraft 的网格坐标系统，每个行为记录绑定一个坐标位置。

建造**档案室/记录板**后，玩家可以查阅种群中 NPC 的行为记录。这构成了反馈回路的物质基础——玩家通过查阅记录发现基因改造的效果，而非依赖系统弹窗。

**视野外 NPC 的记录默认在后台运行，玩家不可见。** 玩家可通过特定方式获取远程记录：

- 植入带有"记录能力"的基因给 NPC——该 NPC 在视野外产生的记录可远程传回
- 建造通讯设施，接收远处携带记录基因的 NPC 的状态
- 使玩家可以通过记录的坐标追踪 NPC 的行踪

## 视野外 NPC 进入视野的规则

世界按坐标网格分块。**每个视野外 NPC 的行为记录包含其当前坐标。** 当玩家坐标接近某个 NPC 最近记录中的坐标时（即 NPC 所在区块进入玩家的加载半径），该 NPC 从后台记录模式切换为全模拟实体。

规则是：记录包含坐标 → 按坐标加载 → 进入视野。不需要活跃度评分或关系判定——NPC 的记录说它在哪，它就在哪。
