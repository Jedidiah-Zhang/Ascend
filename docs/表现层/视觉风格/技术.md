# 视觉风格 — 技术

前端地图渲染由 `scripts/world/map_display.gd`（`MapDisplay`）实现，等轴侧多层 TileMapLayer + spritesheet 纹理。

## 节点结构

```
World (Node2D)
└── MapDisplay (Node2D, map_display.gd)
    ├── MapCamera (Camera2D)          每帧跟随玩家
    ├── ElevationLayers (Node2D)      海拔层容器
    │   ├── Layer0 (TileMapLayer)     海拔 0 层（场景内预置，持有 TileSet）
    │   ├── LayerN (TileMapLayer)     按需动态创建，position.y = -N×16
    │   └── PlayerLayerN              玩家专用层，保证与地形正确排序
    └── Player (Sprite2D)             玩家占位精灵，挂在当前海拔的 PlayerLayer 下
```

## 海拔分层渲染

- 每个海拔值对应一个共享 `TileMapLayer`（`_get_layer(elevation)` 惰性创建），`z_index = elevation`，`position = (0, -elevation × 16)`——高地整体上移 16px/层
- tile 的海拔来源优先级：水面 tile（类型 6/7）强制 0 层 → 后端逐格 `elevation` 数组 → 按地形类型的 `TERRAIN_BANDS` 兜底
- 卸载区块后若某海拔层不再被任何已加载区块使用，该层 `queue_free()` 回收
- 地形类型 → 图集坐标映射：`TERRAIN_TILES`（9 种地形对应 spritesheet 中的固定 atlas 坐标）

## 区块流式加载

以玩家所在 chunk 为中心（chunk = 200×200 tile，与后端 `TILE_MAP_SIZE` 一致）：

```
半径参数（config.gd）
├── INITIAL_VIEW_RADIUS = 2   出生点初始请求 5×5
├── STREAM_MARGIN = 1         维持 3×3 已渲染
├── UNLOAD_RADIUS = 3         超出 7×7 卸载数据
└── MAX_PENDING_TILES = 3     同时在途的 tile 数据请求上限
```

每帧 `stream_chunks_for_viewport()`：

1. 卸载超出 `UNLOAD_RADIUS` 的区块（数据 + 瓦片）；`STREAM_MARGIN`～`UNLOAD_RADIUS` 之间的只擦除瓦片、保留数据缓存
2. 对 3×3 范围内未知区块批量发送 `get_chunks` 请求（元数据）
3. 已有地形数据但未渲染的区块进入放置队列
4. 缺少 tile 数据的区块进入 `_tile_queue`，按 `MAX_PENDING_TILES` 限流逐个请求 `include_tiles=true`

区块状态机：`_pending`（在途）→ `_chunks`（有数据）→ `_being_placed`（放置中）→ `_tiles_loaded`（已渲染）/ `_tiles_cached`（有数据未渲染）。

## 帧预算分批放置/擦除

放置与擦除 40000 格/chunk 的开销分摊到多帧，避免卡顿：

- `_process_queues()` 每帧共享 `PLACE_TIME_BUDGET_US = 5000μs` 时间预算，先放置后擦除
- 每 64 格（`PLACE_BATCH_CHECK_MASK`）检查一次预算，超时保存游标下帧继续
- 卸载时若区块仍在放置队列中则直接取消；擦除完成后回收空海拔层
- 各阶段耗时（stream/place/erase/queue）计入微秒级计时器，供 F3 性能分区显示

## 玩家渲染与移动

- 玩家位置为浮点 tile 坐标（`_player_pos`），移动速度 `PLAYER_SPEED = 80 tile/s`，WASD 方向向量 × delta 本地插值
- `_place_player_at()`：整数格用 `map_to_local` 定位，小数部分沿等轴侧基向量（`_cell_step_x/_cell_step_y`）偏移，实现格间平滑移动
- 玩家海拔变化时在 PlayerLayer 之间重挂载（reparent），保证与地形层的绘制顺序正确
- 相机每帧对齐玩家全局坐标

## 数据查询接口

MapDisplay 缓存的区块数据同时服务调试面板逐格查询：

- `get_elevation_at / get_slope_at`：逐格海拔（米）/ 坡度，来自后端 `elevation` / `slope` 数组
- `get_chunk_temperature / get_chunk_humidity / get_chunk_climate`：区块级年均温 / 年均湿度 / 气候带编码（与后端 `ClimateZone` 枚举一致）
- 无数据时返回哨兵值（-999.0 / -1），调用方显示 "—"
