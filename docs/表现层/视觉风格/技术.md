# 视觉风格 — 技术

前端 3D 渲染由 `scripts/world/map_display_3d.gd`（`MapDisplay3D`）实现，GridMap 地形 + Sprite3D 实体 + 标准 Godot 3D 光照管线。

## 节点结构

```
Main (Node3D, main_world_3d.gd)
├── World (Node3D)
│   ├── Camera3D             正交投影，俯角 60°，锁死旋转
│   ├── DirectionalLight3D   太阳光，方向由游戏时间驱动
│   ├── WorldEnvironment     后处理、雾效、天空色
│   ├── Terrain (Node3D)
│   │   └── ChunkPool (Node3D)
│   │       ├── Chunk_0_0 (GridMap)    每 chunk 一个 GridMap
│   │       ├── Chunk_1_0 (GridMap)
│   │       └── ...
│   ├── Water (MeshInstance3D)         半透明水面 plane，y=0
│   ├── Entities (Node3D)
│   │   ├── Entity_xxx (Node3D)        容器节点
│   │   │   └── Sprite3D               billboard 贴图
│   │   └── ...
│   ├── Caves (Node3D)                 洞穴子场景（按需加载）
│   └── WeatherFX (Node3D)
│       └── RainParticles (GPUParticles3D)
├── TerminalLayer (CanvasLayer, z=100)       不变
│   └── TerminalWidget
└── DebugLayer (CanvasLayer, z=200)          不变
    ├── DebugOverlay
    └── EventLog
```

## 相机配置

```gdscript
# Camera3D 初始化
camera.projection = Camera3D.PROJECTION_ORTHOGONAL
camera.size = 20.0  # 正交半高，缩放时调整此值
camera.position = Vector3(player_x, player_y + OFFSET_Y, player_z + OFFSET_Z)
# 俯角 60°：Y 偏移 / Z 偏移 = tan(60°) ≈ 1.732
```

| 参数 | 值 |
|------|-----|
| 投影 | 正交 (Orthogonal) |
| 俯角 | 60°（camera 高 Z 远，look_at 玩家） |
| 旋转 | 锁死 0°（正北） |
| size（正交半高） | 默认 20，缩放时 ±2 步长，clamp(2, 80) |
| 跟随 | 每帧 camera.position.xy = player.position.xy + offset |

## GridMap 地形

### MeshLibrary

用 Godot 编辑器或脚本构建 MeshLibrary（`.res`），每个地形类型一个 item：

| Item ID | 地形 | Mesh | 说明 |
|---------|------|------|------|
| 0 | 深水 | — | 不占 GridMap，走 Water plane |
| 1 | 浅水 | Cube 1×1×1, 半透明蓝 | 浅滩区域 |
| 2 | 沙滩 | Cube 1×1×1, 沙色顶面 | 顶面土黄 + 侧面沙色 |
| 3 | 平原 | Cube 1×1×1, 草色顶面 | 顶面绿色 + 侧面泥土 |
| 4 | 丘陵 | Cube 1×1×1, 深绿顶面 | 同上，颜色微调 |
| 5 | 岩石 | Cube 1×1×1, 灰色顶面 | 顶面灰色 + 侧面岩石纹理 |
| 6 | 高山 | Cube 1×1×1, 深灰顶面 | 顶面深灰 |
| 7 | 雪峰 | Cube 1×1×1, 白色顶面 | 顶面白色 + 侧面雪岩混合 |
| 8 | 沃土 | Cube 1×1×1, 深棕顶面 | 深色肥沃土壤 |

每个 item 的 cube mesh 使用 `BoxMesh` + `StandardMaterial3D`，不同面贴不同纹理（顶面/侧面可分开 slot）。

### Cell 放置

每 tile 放一格到其海拔高度，利用 GridMap 天然的三维层叠：

```gdscript
func _place_chunk_cells(entry: Dictionary, gridmap: GridMap) -> void:
    var terrain: Array = entry["terrain"]
    var elevation: Array = entry.get("elevation", [])
    var cx: int = entry["cx"]
    var cy: int = entry["cy"]
    var base_x: int = cx * CHUNK_SIZE
    var base_z: int = cy * CHUNK_SIZE

    for i in range(CHUNK_SIZE * CHUNK_SIZE):
        var tz: int = i / CHUNK_SIZE
        var tx: int = i % CHUNK_SIZE
        var elev: int = maxi(0, int(elevation[i])) if i < elevation.size() else 0
        var terrain_id: int = terrain[i]

        # 水面 tile 不放在 GridMap（走 Water plane）
        if terrain_id <= 1:
            continue

        # 每个 tile 放一格在海拔高度；cube 侧面自动形成悬崖
        gridmap.set_cell_item(Vector3i(tx, elev, tz), terrain_id)
```

海拔差导致相邻 tile 的 cube 在不同 Y 层，cube 侧面天然形成陡坡/悬崖视觉——不再需要伪 3D 的 Y 像素偏移。

### 水面

独立于 GridMap 的单层 `PlaneMesh`（或 `QuadMesh`），y=0，覆盖整个世界范围，shader 材质：
- 半透明蓝色（alpha ≈ 0.6）
- 顶点或 UV 动画模拟波纹
- 深度雾效让深水区更暗

### 流式加载

沿用当前 2D 版的流式逻辑 (`_stream_radius_for_zoom` / `_unload_distant_chunks`)：
- 视野内缺失的 chunk → 发 `get_chunks` 请求
- 数据到达后创建 GridMap 节点并逐 cell 放置（分帧预算 `PLACE_TIME_BUDGET_US`）
- 超出视野 → `queue_free()` GridMap
- 中间带（stream_r ~ unload_r）释放 GridMap 保留数据缓存

唯一变化：`TileMapLayer` → `GridMap`，`set_cell(Vector2i, ...)` → `set_cell_item(Vector3i, ...)`。

## 实体渲染（Sprite3D + Billboard）

### 节点结构

```
Entity_<id> (Node3D)
├── Sprite3D (billboard=ENABLED, texture=spritesheet)
└── ShadowDecal / small dark plane (地面阴影指示器)
```

### 位置更新

```gdscript
func place_entity_3d(node: Node3D, world_x: float, world_z: float, elevation_m: float) -> void:
    node.position = Vector3(world_x * CELL_SIZE, elevation_m * CELL_SIZE, world_z * CELL_SIZE)
```

- `CELL_SIZE`：GridMap 每格的世界单位尺寸（默认 1.0）
- 海拔直接映射到 Y 坐标，不需要伪 3D 技巧
- `Sprite3D.billboard = BaseMaterial3D.BILLBOARD_ENABLED`，始终面朝正交相机，无畸变

### 动画

Sprite3D 支持 `texture` 动态替换，配合 `AnimationPlayer` 或脚本逐帧切换实现 2D spritesheet 动画：

```gdscript
# 四向行走（若未来解锁旋转）：N/E/S/W 各一套帧
var walk_frames: Array[Texture2D] = [...]
func _process(_delta):
    frame_index = (frame_index + 1) % walk_frames.size()
    $Sprite3D.texture = walk_frames[frame_index]
```

### 植物 Cross-Plane

```gdscript
# 两片交叉的 Sprite3D，绕 Y 轴 ±45°
var plane1 := Sprite3D.new()
plane1.billboard = BaseMaterial3D.BILLBOARD_DISABLED
plane1.rotation_degrees.y = -45

var plane2 := Sprite3D.new()

```

一体两面，从俯视角度看始终有体积感。

## 光照

### 太阳光（DirectionalLight3D）

```gdscript
# 昼夜循环：根据游戏时间 (0-24h) 计算太阳角度
var sun_angle: float = (game_hour / 24.0) * TAU - PI/2  # 正午在南
sun_light.rotation = Vector3(-PI/4, sun_angle, 0)  # 45° 仰角
sun_light.light_energy = lerp(0.1, 1.0, sun_intensity_factor)
sun_light.light_color = lerp(Color.ORANGE, Color.WHITE, sun_intensity_factor)

# 阴影：低 poly 场景开 PSSM 4-split 足够
sun_light.shadow_enabled = true
sun_light.directional_shadow_mode = DirectionalLight3D.SHADOW_PARALLEL_4_SPLITS
```

- `sun_intensity_factor`：来自后端天气引擎的 `sunshine` / `sunlight_intensity` 参数
- 夜间 → energy 降为 0.05，颜色偏蓝，模拟月光

### 环境光（WorldEnvironment）

```gdscript
var env: Environment = world_environment.environment
env.ambient_light_source = Environment.AMBIENT_SOURCE_COLOR
env.ambient_light_color = lerp(Color(0.02, 0.02, 0.05), Color(0.3, 0.3, 0.35), day_factor)
env.background_mode = Environment.BG_COLOR
env.background_color = Color(0.1, 0.1, 0.12)  # 深灰蓝基调
```

### 局部光源

夜间场景：篝火/火把/营火 → `OmniLight3D`，范围 3-5 单位，暖橙色。性能开销低（场景内通常 <50 个）。

## 遮挡虚化

相机到玩家之间的遮挡物自动半透明：

```gdscript
var _faded_nodes: Dictionary = {}  # Node3D → target_alpha

func _process_occlusion() -> void:
    var space_state := get_world_3d().direct_space_state
    var query := PhysicsRayQueryParameters3D.create(
        camera.global_position, player.global_position)
    query.collide_with_bodies = true
    query.exclude = [player]

    var result := space_state.intersect_ray(query)
    var hit_nodes: Dictionary = {}

    # 命中的遮挡物 → 记录要淡化
    if not result.is_empty():
        var node := result.collider
        while node and node != self:
            hit_nodes[node] = 0.2
            node = node.get_parent()

    # 上一帧淡化但本帧未命中 → 恢复
    for n in _faded_nodes:
        if not hit_nodes.has(n):
            _tween_alpha(n, 1.0)
    for n in hit_nodes:
        if not _faded_nodes.has(n):
            _tween_alpha(n, hit_nodes[n])

    _faded_nodes = hit_nodes

func _tween_alpha(node: Node3D, target: float) -> void:
    var tween := create_tween().set_parallel()
    for child in node.find_children("*", "MeshInstance3D"):
        var mat := child.get_active_material(0) as StandardMaterial3D
        if mat:
            mat.transparency = BaseMaterial3D.TRANSPARENCY_ALPHA
            tween.tween_property(mat, "albedo_color:a", target, 0.15)
```

> 进阶：dithering 溶解 —— 在 `transparency = ALPHA_SCISSOR` 的 shader 里用 screen-door pattern 替代平滑 alpha，视觉更接近 RimWorld 的遮挡风格。实现需自定义 shader，但核心流程同上。

## 洞穴

```gdscript
# 洞穴入口 Area3D 触发
func _on_cave_entrance_body_entered(body: Node3D) -> void:
    cave_roof.visible = false           # 隐藏洞顶
    cave_interior.visible = true         # 显示洞穴内部
    # 可加：地表 illumination 降低，洞穴内加局部光源

func _on_cave_exit_body_exited(body: Node3D) -> void:
    cave_roof.visible = true
    cave_interior.visible = false
```

- 洞穴入口 Area3D 放地表 mesh 层和洞穴入口之间
- 屋顶是一个独立 `MeshInstance3D`（或 GridMap 子集含屋顶层），material 带 `transparency`
- 洞穴内部是独立场景或预制节点，按需加载

## 天气粒子

```gdscript
# 雨
var rain := GPUParticles3D.new()
rain.process_material = preload("res://assets/particles/rain_material.tres")
rain.emitting = weather_data.precipitation_intensity > 0.0
rain.amount_ratio = clamp(weather_data.precipitation_intensity / 10.0, 0.0, 1.0)
rain.position = Vector3(player_x, 30, player_z)  # 跟随玩家
```

粒子材质参数由后端天气数据驱动：强度、风速（粒子水平漂移）、温度（雪/雨切换）。

## 迁移路径概要

| 阶段 | 内容 | 产出 |
|------|------|------|
| 1. 基础场景 | 创建 `Main3D` 场景 + Camera3D + 光照 + WorldEnvironment | 可运行的 3D 空场景 |
| 2. GridMap 地形 | MeshLibrary 构建 + `_place_chunk_cells` + 流式加载适配 | 地形可见 |
| 3. 实体 Sprite3D | EntityLayer 适配 Sprite3D billboard + 位置同步 | 玩家/实体可见 |
| 4. 水面 | Water plane + shader | 水面可见 |
| 5. 天气粒子 | GPUParticles3D 接入天气数据 | 雨雪沙尘可见 |
| 6. 遮挡虚化 | Raycast + alpha tween | 树冠/建筑遮挡处理 |
| 7. 洞穴 | Area3D 触发 + roof toggle | 洞穴可进入 |
| 8. 后处理打磨 | fog / 昼夜色调 / bloom | 视觉完稿 |

每个阶段的 API 层（`Connection` 消息协议、后端数据格式）保持不变，只改渲染消费端。

## 与现有系统的兼容

- **Connection / MsgPack**：不变。`get_chunks` / `entity_snapshot` / `player_state` 等消息格式完全复用
- **Config**：新增 3D 相关常量（`CELL_SIZE`, `CAMERA_PITCH`, `ORTHO_SIZE_MIN/MAX`），其余不变
- **UI**：TerminalLayer / DebugLayer 的 CanvasLayer 在 3D 视口之上，Z 轴隔离，完全不受影响
- **F3 面板**：`get_chunk_stats()` 等接口签名不变，仅在 `MapDisplay3D` 中重新实现
