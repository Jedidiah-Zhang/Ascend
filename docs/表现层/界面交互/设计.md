# 界面交互 — 设计

## 操作方案（键鼠）

| 输入 | 动作 | 处理方 |
|---|---|---|
| WASD / 方向键 | 玩家移动 | 前端插值，每 0.2s 节流上报后端权威实体（player_move） |
| E | 交互 | 发送 `player_interact` 请求 |
| Esc | 菜单 | 发送 `open_menu` 请求（后端未实现） |
| 鼠标滚轮 | 相机缩放（0.15x–4x） | 纯前端 |
| `/` | 切换调试终端 | 纯前端 |
| F3 | 切换调试面板 + 事件日志 | 纯前端 |

设计原则：

- 玩家移动等高频操作前端本地响应，保证手感；后端持有权威玩家实体（壳子版：接受上报，未来加校验），传送等服务端主导的位置变更经 `player_teleported` 事件推送前端吸附
- 游戏逻辑操作（交互、菜单）发请求由后端裁决
- 终端打开时独占键盘输入，相机控制与玩家移动暂停，但网络消息仍持续处理
- 所有调试 UI 均为半透明覆盖层，`mouse_filter = IGNORE` 不拦截游戏操作

## 调试终端

Minecraft 命令行式的全屏半透明控制台，开发调试期的主要交互入口：

- `/` 打开/关闭；输入行 + 滚动输出区（上限 500 行），绿色提示符 `$ `
- 本地指令（`clear`、`tp` 等）经 `register_command` 注册后前端直接处理；其余指令原样转发后端 `terminal_cmd`，输出回显；`help` 同时显示本地与远程帮助
- 指令历史（↑/↓ 翻阅，上限 100 条）、光标编辑（←/→/Home/End/退格/Delete）、PageUp/PageDown 翻页
- 完整指令表见 [终端指令](./终端指令.md)

## F3 调试面板

Minecraft F3 风格，屏幕左上角分区显示，0.25s 节流刷新。当前 9 个分区：

| 分区 | 内容 | 数据来源 |
|---|---|---|
| 性能 | FPS / TPS / MSPT / 各阶段微秒耗时 | 引擎自取 + MapDisplay 计时器 + minute_change 推算 TPS |
| 内存 | 静态/视频内存、节点数 | Performance 单例自取 |
| 时间 | 第 N 天 HH:MM | minute_change 事件 |
| 相机 | 位置、缩放 | 每帧推送 |
| 玩家 | 坐标、区块、海拔 | 每帧推送 |
| 气候 | 温度/湿度/气候带 | chunk 数据 + 天气事件 |
| 天气 | 降水、温/湿/风（含体感描述）、光照强度、日出日落 | `get_weather` API 每 0.5s 轮询 |
| 区块 | 已加载/放置中/缓存/待发送计数 | MapDisplay 统计 |
| 地形 | 逐格海拔、坡度 | chunk 数据（玩家移格时刷新） |

## 事件日志

屏幕右侧与 F3 面板对称的实时事件流（随 F3 一同开关）：

- 显示最近 20 条，新事件插入顶部，0.5s 节流重绘
- 每条格式 `[HH:MM] 事件描述 [区块 x,y]`；日期变更插入 `── 第N天 ──` 分隔线
- **位置过滤**：带位置的天气事件（温度/湿度/风/日照/降水）仅显示玩家周围 3×3 chunk 内的，避免全图事件刷屏
