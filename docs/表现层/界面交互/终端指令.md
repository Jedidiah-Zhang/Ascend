# 终端指令表

调试终端（前端 `/` 键呼出）与后端交互式控制台（`tests/interactive/console.py`）共用同一套远程指令，由后端 `CommandExecutor`（`ascend/terminal/executor.py`）解析执行。本地指令仅存在于 Godot 前端 `TerminalWidget`。

## 本地指令（前端处理，不经网络）

| 指令 | 说明 |
|---|---|
| `clear` | 清空终端输出 |
| `help` / `?` | 显示本地指令帮助，并继续请求后端追加远程指令帮助 |

本地指令通过 `TerminalWidget.register_command(name, handler, help_text)` 注册（模仿 `DebugSection` 注册模式）：

```gdscript
_terminal.register_command(
    "mycmd", _cmd_handler, "mycmd <arg> - 说明文本")

func _cmd_handler(args: PackedStringArray) -> String:
    ...
    return "输出文本"   # 返回文本写入终端，空串无输出
```

## 远程指令（后端 CommandExecutor 执行）

### `status` — 运行状态

| 指令 | 说明 |
|---|---|
| `status` | 运行状态报告：活跃时间/日/时刻/速度/状态 + 世界树统计（tick、事件、publish/trim/订阅者/归档） |

### `time` — 时间控制组

| 指令 | 说明 |
|---|---|
| `time` | 显示当前时间状态（日/时刻/速度/运行状态） |
| `time speed <n>` | 设置时间流速（浮点，`0` 等价暂停；暂停时设 >0 自动恢复） |
| `time pause` | 暂停游戏时间 |
| `time resume` | 恢复游戏时间 |
| `time jump [d]` | 跳转 d 天（默认 1），落地到目标日 06:00，跳过中间事件 |
| `time tick [n]` | 手动推进 n tick（默认 1），忽略暂停与流速 |

### `weather` — 天气查询与强制控制组

坐标 `[cx cy]` 省略时使用出生点 chunk。

| 指令 | 说明 |
|---|---|
| `weather` / `weather status [cx cy]` | 查询该位置当前天气：温度/湿度/风速/日照（时长+强度）/降水/日出日落，附体感描述 |
| `weather set rain <on\|off> [cx cy]` | 强制开启/关闭降雨（开启以均值强度立即生效；事件由下一游戏分钟发布） |
| `weather set <modifier> <on\|off> [cx cy]` | 强制开关特殊气候，`modifier` ∈ `cold_snap` / `heat_wave` / `storm`；气候带天然不出现该修改器的 chunk 也可强制开启 |

### `entity` — 实体生灭调试组（Issue #20）

birth/death 是世界内因果事件，经世界树广播（`entity_born` / `entity_died`），前端实时渲染/移除实体视图。

| 指令 | 说明 |
|---|---|
| `entity` / `entity list` | 列出所有活跃实体（ID 短前缀 / 类型 / 控制者 / 全局坐标 / 层） |
| `entity birth <type> [x y]` | 在指定位置诞生实体，`type` ∈ `creature` / `plant` / `item` / `structure`；坐标缺省为玩家当前位置 |
| `entity death <id前缀>` | 按唯一 ID 前缀（≥4 字符）移除实体；玩家控制的实体受保护，拒绝移除 |

### 独立指令

| 指令 | 说明 |
|---|---|
| `tp [x y]` | 传送玩家到全局 tile 坐标；无参数回出生点。修改后端权威玩家实体，经 `player_teleported` 事件推送前端吸附 |
| `lang [code]` | 查看/切换语言（`zh_CN` / `en_US`） |
| `events [n]` | 显示世界树最近 n 条事件（默认 10） |
| `help` / `?` | 显示远程指令帮助 |
| `q` / `quit` / `exit` | 退出（仅后端交互式控制台有效；前端终端用 `/` 关闭） |

## 扩展接口

- **后端**：`CommandExecutor.register_command(name, handler)`，handler 签名 `(args: list[str]) -> CommandResult`，供 mod 注入新指令。
- **前端**：`TerminalWidget.register_command(name, handler, help_text)`，handler 签名 `(args: PackedStringArray) -> String`。
- 未匹配本地指令的输入原样经 `terminal_cmd` 请求转发后端，输出回显终端。
