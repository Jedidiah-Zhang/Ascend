# 表现层 — 综述

表现层定义玩家感知到的一切：视觉、界面、音频。不包含游戏逻辑，只负责"呈现"。由 Godot 前端（`frontend/`）实现，所有游戏状态来自 Python 后端。

## 子模块关联

```
视觉风格 → 角色外观
    ↓         ↓
  界面交互 ← 音频音效
```

- **视觉风格**定义场景、tile、美术方向（已实现：固定视角 3D 正交相机 + GridMap 地形 + 海拔真实网格 + 区块流式加载）
- **角色外观**定义基因如何驱动角色视觉差异（部位拼接、颜色、纹理）（未实现：当前为 Sprite3D billboard 占位）
- **界面交互**定义 HUD、面板、菜单、操作反馈（已实现：调试终端、F3 调试面板、事件日志、键鼠操作方案）
- **音频音效**定义环境音、动作音、对话音（未实现）

## 前端实现结构

```
frontend/
├── scenes/
│   ├── main.tscn              主场景：World + TerminalLayer + DebugLayer
│   └── world/world.tscn       世界场景：MapDisplay + Camera3D + Terrain + Entities
└── scripts/
    ├── config.gd              集中配置（与后端 ascend/config.py 同步）
    ├── autoload/
    │   ├── connection.gd      Connection 单例：TCP 连接 + 后端进程管理 + 后台解码线程
    │   └── msgpack.gd         MessagePack 编解码
    ├── world/
    │   ├── main_world.gd      主世界编排：输入分发、消息路由、调试数据推送
    │   └── map_display.gd     MapDisplay3D：3D 地形渲染 + 区块流式加载 + 玩家定位 + 遮挡虚化
    └── ui/
        ├── terminal_widget.gd 调试终端（'/' 切换）
        ├── debug_overlay.gd   F3 调试覆盖层（Minecraft F3 风格）
        ├── debug_section.gd   调试分区基类
        ├── debug_sections/    9 个分区：性能/内存/时间/相机/玩家/气候/天气/区块/地形
        └── event_log.gd       右侧事件日志面板（最近 20 条）
```

### 前后端职责划分

前端只做呈现与输入采集，逻辑全部在后端：

- **前端**：3D 地形渲染（GridMap）、实体呈现（Sprite3D billboard）、播放动画、采集输入、显示 UI；相机跟随玩家（固定俯角 60° 正交）
- **后端**：世界生成、时间推进、天气解析、实体管理；前端通过 `request/response` 拉取数据（get_chunks、get_weather、entity_snapshot、player_state、terminal_cmd），通过 `event` 推送接收状态变更（minute_change、temperature_change、precipitation_start、entity_born/died/moved 等）
- **通信**：MessagePack over TCP（4 字节大端长度前缀帧）。前端启动时自动探测端口并按需拉起后端进程

## 与其他大类的接口

- ← **世界框架**：地图、群系、季节的视觉呈现（已接入：地形/海拔/坡度/气候/天气数据经 get_chunks、get_weather 与天气事件流入调试面板与 3D 地形渲染）
- ← **基因系统**：基因效果在角色外观上的可视化（Sprite3D 分层贴图）
- ← **生命个体**：受伤、疾病、需求状态的外观表现
- ← **群体社会**：关系值、记录、管理界面的 UI 呈现
- ← **玩家行动**：所有玩家操作的界面和操作反馈（已接入：移动/交互/菜单/终端指令经 Connection 发往后端）
